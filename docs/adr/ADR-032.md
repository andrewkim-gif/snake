# ADR-032: 좌표 매핑 규칙 (2D → 3D)

## Status
Accepted

## Context
기존 게임 로직은 2D 월드 좌표 (x, y)를 사용합니다. Three.js는 3D 좌표 (x, y, z)를 사용하며, y축이 수직(높이)입니다. 2D 좌표를 3D로 매핑하는 규칙을 정의해야 합니다.

## Decision
**매핑 규칙**: `2D(x, y) → 3D(x, 0, -y)`
- `2D.x → 3D.x` (수평, 변경 없음)
- `2D.y → 3D.z` (수직 → 깊이, **부호 반전**)
- `없음 → 3D.y = 0` (높이, 지면)
- 부호 반전 이유: 2D에서 +y는 아래 방향, Three.js에서 +z는 카메라 쪽. 일관된 시각적 방향을 위해 반전.
- **카메라**: OrthographicCamera, (x+D, D, z+D) 위치에서 (x, 0, z) lookAt → isometric 45° 뷰
- 모든 변환은 `coordinate-mapper.ts` 유틸리티를 통해 수행 (일관성 보장)

## Consequences
- **이점**:
  - 게임 로직 변경 0줄 (2D 좌표 그대로 유지)
  - 서버 변경 0줄 (2D 좌표 기반 직렬화 유지)
  - 단순한 변환 (x,y → x,0,-y)
- **제약**:
  - z축 부호 반전에 주의 필요 (모든 3D 컴포넌트에서 일관 적용)
  - 높이 정보가 없어 모든 엔티티가 y=0 (평면) — v39+에서 높이맵 도입 시 y값 활용
  - 각도 변환 시에도 부호 반전 필요: `angleToRotationY(angle) = -angle`

## Alternatives Considered
1. **2D(x,y) → 3D(x, 0, y)** (부호 반전 없음): 직관적이지만 Three.js +z = 카메라 쪽이라 화면상 방향이 뒤집힘. 제외.
2. **2D(x,y) → 3D(x, y, 0)**: x-y 평면 사용. Three.js 조명/그림자가 y-up 기준이라 비직관적. 제외.
