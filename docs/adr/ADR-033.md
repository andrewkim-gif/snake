# ADR-033: World UI 렌더링 방식 — drei Html

## Status
Accepted

## Context
3D 월드 공간에 데미지 넘버, HP 바, 네임태그, 채팅 버블 등 UI 요소를 표시해야 합니다. 선택지: (A) drei `<Html>` (DOM overlay) vs (B) CSS2DRenderer vs (C) 3D Mesh (Billboard Sprite) vs (D) Canvas Texture.

## Decision
**(A) drei `<Html>` 컴포넌트**를 사용합니다.
- R3F 생태계와 자연스럽게 통합 (JSX로 선언적 사용)
- world-to-screen 프로젝션 자동 처리
- CSS 애니메이션 활용 (GPU accelerated float-up, fade 등)
- DOM Pool 패턴으로 성능 최적화 (max 88 DOM 요소)

## Consequences
- **이점**:
  - CSS 기반 스타일링 → 디자인 자유도 높음
  - 기존 React 컴포넌트 패턴과 동일한 개발 경험
  - occlude, distanceFactor 등 편의 props 제공
  - 텍스트 렌더링 품질 우수 (시스템 폰트 사용)
- **제약**:
  - 다수 DOM 요소 시 레이아웃 리플로우 성능 우려 (→ DOM Pool로 완화)
  - z-index 관리 필요 (HUD overlay와 충돌 방지)
  - 88 DOM 요소 상한 설정 (데미지 40 + HP 30 + 이름 15 + 채팅 3)
- **탈출구**: Phase 6 벤치마크에서 성능 문제 시 CSS2DRenderer로 전환 가능

## Alternatives Considered
1. **(B) CSS2DRenderer**: Three.js 내장. 비슷한 접근이지만 R3F 생태계 외부. drei 호환성 이슈. 2순위 대안.
2. **(C) 3D Billboard Sprite**: 텍스트 렌더링 품질 낮음 (Canvas texture). 복잡한 UI 구현 어려움. 제외.
3. **(D) Canvas Texture**: GPU 메모리 사용 증가. 동적 텍스트 업데이트 비용 높음. 제외.
