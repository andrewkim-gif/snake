# ADR-030: Frame Swap 캐릭터 애니메이션 방식

## Status
Accepted

## Context
Voxel 캐릭터의 애니메이션 방식을 결정해야 합니다. 선택지: Frame Swap (프레임별 geometry 교체), Skeletal Animation (bone 기반), Morph Target (형태 보간), Shader Bone (GPU 기반).

## Decision
**Frame Swap 방식**을 1순위로 채택합니다.
- 액션별 프레임 수: idle(2f), walk(4f), hit(2f), death(3f) = 총 11프레임
- 프레임당 별도 .glb 파일 (geometry만 교체, material 공유)
- useFrame에서 elapsed time 기반 프레임 인덱스 계산 → `mesh.geometry = frames[index]`

## Consequences
- **이점**:
  - 복셀 미학 완벽 유지 (프레임 간 보간 없음 = 의도적 뚝뚝 끊기는 느낌)
  - 구현 단순 (geometry swap만)
  - MagicaVoxel에서 프레임별 직접 제작 → Blender 변환 워크플로우와 자연스럽게 호환
  - CPU 부하 낮음 (bone 계산 없음)
- **제약**:
  - 파일 수 증가 (9 클래스 × 11 프레임 = 99 .glb)
  - 에셋 용량 증가 (~20KB × 99 = ~2MB)
  - 부드러운 전환 불가 (디자인 의도)

## Alternatives Considered
1. **Skeletal Animation**: 파일 크기 작지만 CPU 부하 높음 (100+ 동시 엔티티에서 병목). 부드러운 전환은 복셀 미학과 불일치. 제외.
2. **Morph Target**: 보간 시 비복셀 느낌. 제외.
3. **Shader Bone (GPU)**: 1 draw call로 대량 엔티티 가능. 고급 기술로 v39+에서 검토. 2순위 대안.
