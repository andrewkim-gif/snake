# ADR-034: 2D/3D 듀얼 모드 아키텍처

## Status
Accepted

## Context
Canvas 2D 렌더링을 Three.js 3D로 교체하되, 기존 2D 모드를 유지하여 사용자가 선택할 수 있어야 합니다. WebGL 미지원 기기에서는 자동 2D fallback이 필요합니다.

## Decision
**MatrixApp에서 renderMode 분기**합니다.
- `renderMode: '2d' | '3d'` 상태를 MatrixApp에서 관리
- `'2d'` → 기존 MatrixCanvas.tsx 마운트
- `'3d'` → 신규 MatrixScene.tsx 마운트
- 전환 시 200ms fade transition (WebGL context conflict 방지)
- 두 컴포넌트 모두 동일한 `useGameRefs()`와 `useGameLoop()` 사용
- localStorage에 preference 저장
- WebGL 미지원 감지 시 자동 '2d' fallback

## Consequences
- **이점**:
  - 기존 2D 코드 100% 보존 (삭제 없음)
  - 사용자 선택권 제공 (성능/미학 선호에 따라)
  - WebGL 미지원 기기에서도 게임 플레이 가능
  - A/B 테스트 용이 (2D vs 3D 성능/사용자 반응 비교)
- **제약**:
  - 번들에 2D + 3D 코드 모두 포함 (→ dynamic import로 완화)
  - 두 렌더러 모두 유지보수 필요
  - 전환 중 200ms 블랙스크린 (검증된 패턴이지만 UX 약간 불편)
- **핵심 보장**: 게임 로직(update)은 Worker에서 연속 실행되므로, 모드 전환 중에도 게임 상태 유지 (적 이동, 투사체 진행 등 중단 없음)

## Alternatives Considered
1. **3D only (2D 제거)**: 단순하지만 WebGL 미지원 기기에서 게임 불가. NFR-7 위반. 제외.
2. **동시 마운트 (visibility toggle)**: 두 Canvas를 동시에 마운트하고 display:none 토글. WebGL 리소스 낭비. 제외.
3. **Canvas 2D on WebGL (2D → WebGL 2D)**: PixiJS 등으로 2D를 WebGL 기반으로 전환. 기존 코드 대규모 수정 필요. 목적에 맞지 않음. 제외.
