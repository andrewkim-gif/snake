# ADR-021: Interpolation-Only Client (No Client-Side Prediction)

## Status

Accepted

## Context

The server runs at 20Hz while the client renders at 60fps. To bridge this gap, the client needs a strategy to smooth visual movement between server state updates.

Options:
1. **Interpolation only**: Client smoothly transitions between the two most recent server states (50ms visual delay)
2. **Client-side prediction + server reconciliation**: Client predicts local player movement immediately, reconciles when server state arrives
3. **Extrapolation**: Client predicts all entities' future positions based on velocity vectors

## Decision

**Option 1: Interpolation only.**

```typescript
// Every render frame:
const alpha = (Date.now() - lastStateTime) / 50; // 50ms = 1 tick
const renderX = lerp(prevState.x, currState.x, clamp(alpha, 0, 1));
const renderY = lerp(prevState.y, currState.y, clamp(alpha, 0, 1));
```

The client always renders positions between two confirmed server states. This introduces a fixed ~50ms visual delay but guarantees:
- No rubber-banding (extrapolation can overshoot)
- No desync corrections (prediction can diverge)
- Perfectly smooth movement at 60fps

## Consequences

**Positive:**
- Zero complexity on client side (no physics simulation, no reconciliation logic)
- Guaranteed visual consistency -- what you see is what the server computed
- Identical visual experience for all connection qualities
- Simpler debugging (client state always matches server)
- Auto-combat game (not FPS) -- 50ms delay on aura combat is imperceptible

**Negative:**
- 50ms visual delay on own agent's movement (1 tick behind server)
- High-latency connections (150ms+ for Korean users) feel slightly sluggish
- Boost/dash initiation has ~75-125ms visual delay (latency + 1 tick)

**Why this is acceptable for Agent Survivor:**
- **Auto-combat**: Aura DPS is continuous, not reaction-based. 50ms doesn't affect outcome.
- **Steering (not aiming)**: Player controls direction angle, not precise targeting.
- **Dash is a strategic choice, not a twitch mechanic**: 100ms delay on dash initiation is acceptable.
- **AI agents don't care about visual delay**: They operate on server state directly.

## Alternatives Considered

- **Option 2 (client prediction)**: Rejected for Phase 1. Adds ~500 lines of client-side physics simulation, prediction rollback, and reconciliation. Can be revisited in Phase 3 if Korean user feedback indicates unacceptable latency.
- **Option 3 (extrapolation)**: Rejected -- causes rubber-banding when entities change direction. Particularly bad for the erratic movement patterns of bot AI and other players.
