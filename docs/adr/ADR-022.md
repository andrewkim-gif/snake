# ADR-022: Bot AI -- Priority-Based Behavior Tree

## Status

Accepted

## Context

The game needs AI bots to fill rooms (target: 15 bots per room) and provide engaging PvE content. The bot AI system must:
1. Run within the 2ms tick budget (no external API calls)
2. Create diverse gameplay (not all bots doing the same thing)
3. Scale difficulty over the round duration
4. Make meaningful upgrade choices (following build paths)

Options:
1. **Simple state machine**: Finite states (wander, chase, flee) with transitions
2. **Priority-based behavior tree**: Hierarchical priority evaluation each tick
3. **Utility AI**: Score-based action selection (highest utility wins)
4. **ML/Reinforcement learning**: Trained neural network models

## Decision

**Option 2: Priority-based behavior tree.**

Each tick, the bot evaluates conditions from highest to lowest priority:
```
1. Danger Avoidance (flee from stronger enemies when low HP)
2. Level-Up Response (choose upgrade based on build path)
3. Kill Opportunity (chase weaker enemies)
4. Orb Collection (move toward high-value orb clusters)
5. Map Object Usage (heal at springs when damaged)
6. Wander (random exploration, stay in bounds)
```

The first condition that matches executes its action. This creates naturally prioritized behavior without complex state transition logic.

## Consequences

**Positive:**
- Predictable performance: O(6) condition checks per bot per tick = ~0.1ms for 15 bots
- Easy to tune: adjust condition thresholds and priority order
- Diverse behavior: 5 build paths create 5 distinct playstyles
- Difficulty scaling: `botAggressiveness` multiplier adjusts kill-seeking behavior over time
- Debuggable: can log which behavior node activated

**Negative:**
- Bots are ultimately predictable (experienced players learn patterns)
- No emergent strategy (compared to ML approaches)
- Build path choices are greedy (no long-term planning for synergies)

**Why not ML (Option 4):**
- ML models require training infrastructure, inference runtime, and model versioning
- 15 bots x 50 rooms = 750 simultaneous model inferences at 20Hz = 15,000/s -- too expensive
- Behavior tree achieves "good enough" for PvE content purpose
- ML can be explored in Phase 5+ for premium "smart bot" tiers

## Alternatives Considered

- **Option 1 (state machine)**: Rejected -- state explosion with 6+ behaviors and transitions. Behavior tree naturally handles priority without explicit transitions.
- **Option 3 (utility AI)**: Viable alternative but over-engineered for current needs. Utility scoring adds complexity without proportional behavior improvement for bot-level AI.
- **Option 4 (ML/RL)**: Deferred to Phase 5+. See "Why not ML" above.
