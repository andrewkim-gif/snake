# ADR-031: 게임 루프 추출 전략

## Status
Accepted

## Context
MatrixCanvas.tsx의 `update()` 함수 (~1,150줄)는 모든 게임 로직을 포함합니다. 3D 렌더링으로 전환하려면 이 로직을 렌더러와 분리해야 합니다. 분리 방식: (A) useGameLoop 훅 추출 vs (B) update() 함수만 export vs (C) GameEngine 클래스 분리.

## Decision
**(A) useGameLoop 커스텀 훅으로 추출**합니다.
- MatrixCanvas의 update() 함수를 `apps/web/lib/matrix/hooks/useGameLoop.ts`로 이동
- 내부 로직 변경 0줄 — 외부 의존성(refs, callbacks)만 파라미터로 전달
- Web Worker 메시지 수신 로직도 함께 포함
- useGameLoop은 R3F useFrame과 독립적으로 실행 (Worker 기반 60Hz)

## Consequences
- **이점**:
  - 로직 변경 0줄 보장 → 기존 밸런스 100% 보존
  - MatrixCanvas(2D)와 MatrixScene(3D) 모두 동일 훅 사용 → 코드 중복 제거
  - 테스트 용이 (훅 단위 검증)
- **제약**:
  - 훅 파라미터가 복잡 (GameLoopDeps 인터페이스에 ~20개 필드)
  - 추출 후 기존 MatrixCanvas에서 리그레션 테스트 필수
- **검증**: 추출 후 2D 모드에서 리그레션 0 확인 → 3D 모드에서 동일 훅 사용

## Alternatives Considered
1. **(B) update() 함수만 export**: 단순하지만 Worker 연결 로직이 분리되지 않음. 훅 패턴이 React 생태계에 더 적합.
2. **(C) GameEngine 클래스**: OOP 스타일. React 훅 생태계와 어울리지 않음. refs 접근이 복잡해짐. 제외.
