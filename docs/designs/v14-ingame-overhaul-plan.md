# PLAN: v14 — In-Game Total Overhaul (AI World War: Deathmatch)

> Megabonk 영감의 자동전투 데스매치 + 국가 지배 문명 시뮬레이션

## 1. 개요

현재 AI World War의 인게임은 5분 단판 서바이벌이다. 이를 **Megabonk 스타일의 자동전투 데스매치**로
완전히 재설계하고, 그 위에 **국가 지배 → 문명 관리 → 전쟁 시스템**의 메타게임 레이어를 얹는다.

**핵심 변경:**
- 1-life 서바이벌 → **리스폰 데스매치** (5분 평화 + 5분 전투, 10분 에포크)
- 5분 단판 → **1시간 지배 평가** → 24시간 지속 → 1주 헤게모니
- 단순 톰 업그레이드 → **10종 무기 + 10종 패시브 + 시너지 콤보** 풀 스킬 트리
- 스킨 선택 → **로비 캐릭터 크리에이터 기반 랜덤 생성** + 국적 설정
- 국가별 아레나 50명 → 전투 결과가 **지정학적 지배**에 직결
- 지배국이 **종교, 언어, 정치 체제, 세율** 등 정책 변경 가능
- 국민 행복지수, 출산율, GDP 등 **문명 지표** 실시간 가시화
- 지구본에서 **지배 현황 + 전쟁 연출** 실시간 표시

**에포크 상세:**

| 시간 | 이벤트 | 설명 |
|------|--------|------|
| 0:00 | 에포크 시작 | 전원 리스폰, 레벨 유지, 무기/패시브 유지 |
| 0:00~5:00 | **평화 페이즈** | PvP OFF, 오브 파밍, NPC 몬스터, 전략 포인트 점령 |
| 5:00 | **전투 개시** | 경고 사이렌 + 3초 카운트다운, PvP ON |

| 9:00 | **최후의 1분** | 축소 가속, 혼전 유도 |
| 10:00 | **에포크 종료** | 스코어보드, MVP 선정, 국가별 점수 집계 |

**에포크 간 전환 (10초):**
- 결과 오버레이 (5초): 개인 성적 + 국가별 점수 + MVP
- 준비 시간 (5초): 다음 에포크 카운트다운
- 레벨/빌드는 **에포크 간 유지** (1시간 세션 동안)
- 1시간(6에포크) 후 **전체 리셋** + 지배 평가

**리스폰 규칙:**
- 사망 → 3초 대기 (사망 화면 + 킬캠) → 랜덤 위치 리스폰
- 리스폰 후 5초 무적 (반투명 + 골드 글로우)
- 리스폰 시 레벨/빌드 유지, HP 100% 회복
- 리스폰 페널티: 2초간 이동속도 -30% (위치 선정 불이익)
- 에포크당 리스폰 무제한

### 4.2 전투 시스템 (Megabonk 영감)

**핵심 원칙**: 플레이어는 **이동과 포지셔닝**만 조작. 무기는 **자동발사**.

```
현재 시스템                      새 시스템 (Megabonk 스타일)
─────────────                   ─────────────────────────
오라 전투 (60px 범위)     →     무기별 자동발사 패턴
단일 DPS 수치             →     무기마다 고유 사거리/패턴/쿨다운
부스트(대시) = 유일한 기술 →     무기 + 패시브 + 시너지 콤보
Mass = HP               →     HP + DPS + 방어력 분리
```

**전투 메카닉:**

| 요소 | 설계 |
|------|------|
| **HP** | 기본 100, 레벨업 시 +10, Vigor 패시브로 추가 |
| **자동공격** | 보유 무기가 각자의 타이밍/패턴으로 자동 발사 |
| **무기 슬롯** | 최대 5개 동시 장착 (레벨업으로 획득) |
| **이동** | WASD/조이스틱, 기본 150px/s, Swift로 증가 |
| **대시** | Spacebar, 300px/s 1초 돌진, 5초 쿨다운 |
| **크리티컬** | 기본 5%, Precision 패시브로 증가, 200% 데미지 |
| **넉백** | 피격 시 10px 밀림, 일부 무기 추가 넉백 |
| **DOT** | 화염/독 무기의 지속 데미지 (2초간 틱당 피해) |

**데미지 공식:**
```
FinalDamage = BaseDmg × (1 + Fury×0.15) × CritMult × TerrainMod
             - DefenderHP × IronSkin × 0.12
```

### 4.5 캐릭터 시스템 리뉴얼

**변경 사항:**
```
현재                              신규
─────                            ─────
스킨 선택 (24종)          →      CharacterCreator 기반 랜덤 생성
skinId 기반 컬러링         →      CubelingAppearance 전체 적용

로비와 게임 캐릭터 별개    →      로비 = 게임 동일 외형
```

**랜덤 캐릭터 생성 흐름:**
1. 유저가 이름 입력
2. **국적 선택** (195개국 중 택 1, 검색 가능)
3. "RANDOMIZE" 버튼 → CharacterCreator의 랜덤 생성 로직 호출
4. 결과: 얼굴, 눈, 입, 페이스 마크, 모자, 등 아이템, 보디 오버레이, 악세서리 모두 랜덤
5. "REROLL" 가능 (무한), 마음에 들면 "CONFIRM"
6. 확정된 외형이 게임 내 3D 캐릭터에 동일하게 적용

**국적 시스템:**

- 국적 = 전투 결과가 반영되는 소속국
- 국적은 세션 중 변경 불가 (다음 세션에서 변경 가능)
- 인게임에서 머리 위에 국기 아이콘 표시 (16×16)
- 같은 국적 플레이어끼리는 이름이 **초록색** (아군 식별)

### 4.6 국가 지배 시스템 (Domination)

**지배 계층 구조:**

```
10분 에포크 → 국가별 점수 집계
    ↓ ×6
1시간 → ★ DOMINATION EVALUATION ★ → 지배국 변경
    ↓ ×24
1일 지속 지배 → ★ SOVEREIGNTY (통치권) ★
    ↓ ×7
1주 헤게모니 → ★ HEGEMONY ★ → 정책 변경 가능
```

**에포크 점수 계산 (10분당):**
```
NationScore[country] = Σ (각 소속 플레이어의 점수)

PlayerEpochScore = (Kills × 10) + (Assists × 4) + (Level × 5)
                 + (SurvivalMinutes × 2) + (ObjectivePoints × 3)
```

**1시간 지배 평가 (6 에포크 후):**
```
DominationScore[country] = Σ(epoch1~6 NationScore[country])

지배국 결정:
1. 가장 높은 DominationScore를 가진 국가
2. 동점 시: 총 킬 수 > 총 레벨 합 > 먼저 도달
3. 최소 의미 있는 점수 (MinDominationThreshold = 100) 이상이어야 함
4. 현 지배국은 방어 보너스 +10% (방어자 이점)
```

**지배국 전환:**
- 지배국 변경 시 글로브에 즉시 색상 반영
- 알림: "🇰🇷 KOREA now dominates 🇯🇵 JAPAN!"
- 전환 시 15분간 방어 보너스 +20% (빈번한 전환 방지)

**지속 지배 (24시간):**
- 같은 국가가 24시간 연속 지배 유지 → **통치권(Sovereignty)** 획득
- 통치권 = 해당 국가에서 소속 플레이어들에게 버프:
  - +10% XP 획득
  - +5% 이동속도
  - 전략 포인트 점령 속도 +20%
- 통치권 UI: 국기 + "SOVEREIGNTY" 배지

**헤게모니 (1주 지속):**
- 통치권을 7일 연속 유지 → **헤게모니(Hegemony)** 달성
- 헤게모니 = **정책 변경 권한** 해금 (§4.7 참조)
- 글로브에서 해당 국가가 **지배국 색상으로 강하게 빛남**
- 매주 월요일 00:00 UTC 기준 평가

| 1 | 캐릭터 생성 + 국적 선택 가이드 | 첫 접속 |
| 2 | 이동 & 오브 수집 (평화 페이즈) | 첫 게임 입장 |
| 3 | 무기 자동발사 + 레벨업 선택 안내 | 첫 레벨업 |
| 4 | 데스매치 전투 흐름 | 첫 전투 페이즈 |
| 5 | 지배 시스템 개요 | 첫 에포크 종료 |
| 6 | 글로브 & 국가 정보 안내 | 첫 아레나 탈출 |

### 5.2 매치메이킹 & 밸런싱

- **국가 아레나 입장**: 글로브에서 원하는 국가 클릭 → 입장 (50명 제한)
- **오버플로우**: 50명 초과 시 대기열, 30초마다 빈 자리 체크
- **봇 필링**: 인원 부족 시 AI 봇으로 채움 (최소 10명 보장)
- **봇 국적**: 해당 아레나 국가의 주변국 국적으로 랜덤 배정
- **밸런스**: 전투 페이즈 시작 시 레벨 하한선 3 (평화 페이즈 파밍 최소치)

### 5.3 거점 점령 (전략 포인트)

각 아레나에 3개의 전략 포인트 배치:
- **자원 거점**: 점령 시 소속 국적 플레이어 +5 XP/s
- **버프 거점**: 점령 시 소속 국적 플레이어 +10% DMG
- **힐링 거점**: 점령 시 소속 국적 플레이어 +3 HP/s

점령 규칙:
- 5초간 거점 범위(100px) 내 체류 시 점령 시작
- 적 플레이어가 범위 내 있으면 점령 중단 (경합 상태)
- 점령 완료 후 2분간 유지, 이후 중립화 (재점령 필요)
- 전쟁 시: 점령 포인트가 전쟁 목표로 전환

### 5.4 일일 도전 & 업적

**일일 도전 (매일 3개 랜덤):**
- "10킬 달성", "5 에포크 생존", "전략 포인트 3개 점령"
- 보상: 경험치 부스트, 코스메틱 코인

**업적 (영구):**
- "첫 헤게모니", "100킬 달성", "모든 무기 Lv5 달성"
- 보상: 칭호, 특수 이모지, 프로필 뱃지

### 5.5 관전 시스템

- 아레나 풀 시 관전 모드로 입장 가능
- 자유 카메라 + 특정 플레이어 팔로우
- 전쟁 중 관전자 수 표시 (흥행 지표)

### 5.6 날씨 & 환경 (P3)

- 각 국가의 실제 기후 기반 환경 효과 (선택적)
- 사막: -10% 이동속도, 열기 DOT
- 설원: -20% 시야, 얼음 바닥 미끄러짐
- 열대: +10% HP 재생, 시야 제한 (정글)
- 도시: 장애물 다수, 은신처 활용

## 6. 기술 방향

### 서버 (Go)
- **기존 유지**: Go 게임 서버, WebSocket (gorilla/websocket), 20Hz 틱 루프
- **확장**: Arena → CountryArena (195개국 인스턴스, lazy init)
- **신규**: 무기 시스템 (weapon.go), 리스폰 매니저 (respawn.go), 에포크 관리 (epoch.go)
- **신규**: 지배 평가 엔진 (domination.go), 전쟁 상태 머신 (war.go)
- **신규**: 문명 시뮬레이션 (civilization.go — 정책 효과 + 지표 계산)
- **데이터**: Redis (국가 지표 캐시) 또는 인메모리 (초기 구현)

### 클라이언트 (Next.js + R3F)
- **기존 유지**: React Three Fiber, InstancedMesh, cubeling 캐릭터
- **변경**: AgentInstances 확장 (국기 아이콘, 외형 데이터 반영)
- **신규**: WeaponRenderer (무기 이펙트 비주얼), DamageNumbers (떠오르는 데미지 숫자)
- **신규**: LevelUpUI (3개 선택지 오버레이), EpochHUD (타이머 + 페이즈 표시)
- **신규**: GlobeOverlay (지배 색상 + 전쟁 연출 + 국가 패널 확장)
- **신규**: CivilizationPanel (정책 관리 + 지표 대시보드)

### 네트워크 프로토콜 확장
```
신규 이벤트:
  C→S: select_nationality, select_upgrade, declare_war, set_policy
  S→C: epoch_start, epoch_end, war_phase_start, war_phase_end,
       respawn, level_up_choices, weapon_acquired, damage_dealt,
       domination_update, war_declared, war_ended, policy_changed,
       nation_stats_update, capture_point_update
```

### 스타일/디자인
- 기존 Dark Tactical 테마 유지
- 전쟁 연출: 붉은 톤 오버레이 + 파티클
- HUD: 미니멀 + 정보 밀도 높은 전술 스타일

**핵심 데이터 흐름:**
1. **게임 루프** (20Hz): Arena tick → 무기 발사 → 충돌 → 데미지 → 사망/리스폰 → 스코어
2. **에포크 루프** (10분): 평화→전투→축소→종료 → 점수 집계 → 다음 에포크
3. **지배 루프** (1시간): 6에포크 종료 → DominationEngine 평가 → 지배국 갱신 → 글로브 반영
4. **문명 루프** (비동기): 정책 변경 → CivilizationEngine → 지표 재계산 → 클라이언트 알림
5. **전쟁 루프** (이벤트): 선전포고 → 준비기 → 활성 → 종료/항복 → 보상/페널티

## 8. 리스크

| 리스크 | 영향 | 발생 확률 | 완화 전략 |
|--------|------|---------|----------|
| **195개 아레나 동시 운영 메모리** | 서버 메모리 폭증 | 높음 | Lazy init (활성 유저 있는 아레나만), 비활성 아레나 1분 후 해제 |
| **50명 전투 + 무기 이펙트 부하** | 클라이언트 FPS 저하 | 중간 | 무기 이펙트 LOD, 원거리 에이전트 이펙트 간소화, 파티클 풀 |
| **밸런스 문제 (10무기×10패시브)** | 특정 빌드 OP | 높음 | 시뮬레이션 밸런스 테스트, 핫픽스 밸런스 패치 구조 |
| **에포크 전환 시 네트워크 스파이크** | 끊김/지연 | 중간 | 에포크 전환 데이터 경량화, 델타 전송 |
| **전쟁 크로스-아레나 복잡도** | 버그 증가 | 높음 | 전쟁을 별도 아레나로 분리 (Phase 7 후반에 구현) |
| **문명 시뮬레이션 복잡도** | 개발 지연 | 중간 | 기본 지표만 먼저 구현, 피드백 루프는 점진 추가 |
| **인게임 UX 과부하** | 신규 유저 이탈 | 중간 | 튜토리얼 5단계 + 평화 페이즈 학습 시간 |
| **기존 코드와의 충돌** | 마이그레이션 난이도 | 높음 | Phase 0에서 기존 시스템 호환 레이어 구축 |
