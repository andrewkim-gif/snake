# PLAN: v37 — 인게임 전면 고도화 (토큰 이코노미 통합)

> v37 — 스킬 리디자인 + 이펙트 리디자인 + UI/UX 리디자인 + 아이콘 리디자인
> 토큰 이코노미(v36)와 결합한 인게임 전투 경험의 완전한 재설계
> 작성일: 2026-03-10

## 1. 개요

### 1.1 왜 전면 고도화인가

현재 Matrix 게임의 스킬 시스템은 **개발자 밈(meme)** 테마로 구축되어 있다:
- 무기명이 "손코딩", "키보드 펀치", "API 호출", "Git Push" 등 — **코딩 유머**
- 아이콘이 `/assets/skills/` 폴더의 픽셀아트 PNG — 일관성 없고, 해상도 들쭉날쭉
- 이펙트가 "JSON 투사체", "NPM 패키지 폭격", "Git 커밋 메시지" 등 — **세계관과 불일치**
- UI가 기능 중심으로 쌓여옴 — 디자인 철학의 통일성 부재

**AI World War**는 **팩션(진영)이 국가를 점령하는 전쟁** + 토큰 이코노미 + 세계 지배의 진지한 테마를 가진 게임이다.
Hub/World 레이어에서 플레이어는 **팩션에 소속**되어 **국가 아레나에서 전투**하며, 승리한 팩션이 해당 국가의 **주권(Sovereignty)**을 획득한다.
인게임 전투의 비주얼/UX가 이 **팩션-국가 이중 구조** 세계관에 맞게 **완전히 재설계**되어야 한다.

> **세계 모델**: 플레이어 → 팩션 소속 → 국가 아레나 진입 → Nation Score 기여 → 최고 점수 팩션이 해당 국가 주권 획득

### 1.2 v36 토큰 이코노미와의 결합

v36이 설계한 3-Tier 재화 체계(Match Gold → Account RP → On-chain $AWW)를 인게임에서 **체감**할 수 있어야 한다:
- 스킬이 강해질수록 **시각적으로 더 화려해지면서** 동시에 **더 많은 Gold/Score를 생성**
- 레벨업 선택이 단순한 스텟 강화가 아니라 **경제적 의사결정** (전투 효율 vs Gold 수익률)
- UI가 실시간으로 **토큰 획득 파이프라인을 피드백** (골드 획득 → 점수 누적 → 보상 예측)

### 1.3 핵심 변경 영역

| 영역 | 현재 | 목표 |
|------|------|------|
| **스킬 체계** | 25 무기 (코딩 밈 테마) | 20 무기 (군사/전략 테마) + 6 패시브 + 4 궁극기 |
| **스킬 이펙트** | 코딩 비주얼 (JSON, Git, NPM) | 군사/에너지 비주얼 (포탄, 레이저, 방어막) |
| **스킬 아이콘** | 픽셀아트 PNG (불일치) | SVG 플랫 아이콘 (통일된 디자인 시스템) |
| **UI/UX** | 기능별 누적 (20개 컴포넌트) | 통합 전투 HUD + 경제 피드백 레이어 |
| **경제 연결** | 없음 | Gold 실시간 표시 + 전장 상점 + 보상 예측 |

★ `nation_loyalty`는 **신규 패시브**. 토큰 경제의 핵심 — Nation Score가 높을수록 에포크 토큰 보상이 커지므로, 이 패시브는 **장기 투자형** 선택이 된다. 전투 효율은 올라가지 않지만 보상이 커진다.

이로써 레벨업 선택에 **전투 효율 vs 경제 효율** 트레이드오프가 생긴다:
- 전투형: STEEL/TERRITORY 무기 → 더 빨리 킬 → 현재 에포크 생존율 ↑
- 경제형: 전리품/국가 충성 패시브 → 더 많은 Gold/Score → 에포크 종료 후 토큰 보상 ↑

### 2.5 진화 시스템 리디자인

기존 progressive-tree의 진화 시스템을 경제 통합형으로 재설계한다.

**현재 진화 구조** (유지):
```
Tier 1 (Lv1~5)  → 기본 무기, 스텟 점진 증가
Tier 2 (Lv6~10) → 스텟 가속 + 보조 효과 추가
Tier 3 (Lv11)   → EVOLUTION — 비주얼 변화 + 대폭 강화
Tier 4 (Lv20)   → ULTIMATE — 최종 형태, 특수 효과
```

**새로운 진화 연출**:

| 단계 | 현재 연출 | 새 연출 | 경제 연결 |
|------|----------|---------|----------|
| Tier 2 도달 | 없음 | 무기 아이콘 테두리 발광 (카테고리 컬러) | Gold 드롭률 +10% |
| Tier 3 진화 | 아이콘 변경 | **진화 컷씬 0.5초** — 구→신 모프 애니메이션 + 파티클 폭발 | 진화 보너스 Gold 500 즉시 지급 |
| Tier 4 궁극 | 아이콘 변경 | **궁극 컷씬 1초** — 화면 플래시 + 무기 회전 + 타이틀 표시 | Nation Score +1000 즉시 |

**진화 파티클 시스템** (카테고리별):
- STEEL: 금속 파편 + 불꽃 스파크 (레드/오렌지)
- TERRITORY: 에너지 링 + 전기 아크 (블루/화이트)
- ALLIANCE: 체인 연결 + 보라색 오라 폭발 (퍼플)
- SOVEREIGNTY: 육각형 실드 조각 + 녹색 파동 (그린)
- MORALE: 별 파티클 + 충격파 링 (시안)

**시너지 진화** (2개 이상 카테고리 Tier 3 달성 시):
- 시너지 스킬이 잠금 해제될 때 양쪽 카테고리 컬러가 혼합된 **듀얼 컬러 파티클**
- 잠금 해제 알림: HUD 상단에 "SYNERGY UNLOCKED: [스킬명]" 배너 (3초)

**경제적 진화 인센티브**:
- Tier 2: 해당 무기 킬 시 Gold +10%
- Tier 3: 해당 무기 킬 시 Gold +25% + 진화 축하금 500G
- Tier 4: 해당 무기 킬 시 Gold +50% + Nation Score 보너스
- 전 무기 Tier 3 이상: "무기고 마스터" 칭호 + 매치 Gold 총합 +20%

## 3. 스킬 이펙트 전면 리디자인

### 3.1 이펙트 디자인 원칙

모든 스킬 이펙트는 다음 4대 원칙을 따른다:

**원칙 1: 가독성 우선 (Readability First)**
- 적 HP가 많은 화면에서도 **내 무기 이펙트가 즉시 인지**되어야 함
- 이펙트의 히트박스 = 시각적 크기 (보이는 만큼 맞는다)
- 카테고리별 **고유 컬러 + 고유 형태**로 0.1초 내 식별 가능
- 배경(타일맵)과 확실히 분리되는 발광(glow) + 고대비 처리

**원칙 2: 레벨 스케일링 (Visual Progression)**
- Lv1~5: 심플한 단색 이펙트, 작은 크기, 파티클 최소
- Lv6~10: 크기 증가, 보조 컬러 추가, 잔상(trail) 등장
- Lv11 (진화): 형태 변화 + 새로운 파티클 세트 + 글로우 레이어
- Lv12~19: 점진적 밀도/속도 증가
- Lv20 (궁극): 최대 비주얼 — 멀티 레이어, 풀 파티클, 카메라 쉐이크

**원칙 3: 경제 시각 피드백 (Economy Feedback)**
- Gold 생성 킬 시: 적 사망 위치에서 **금색 코인 파티클** 튀어나옴
- 크리티컬 킬 시: 코인 파티클 + **"CRITICAL GOLD +150"** 플로팅 텍스트
- 콤보 보너스 킬 시: 콤보 배율에 따라 코인 크기/개수 증가
- Nation Score 기여 킬 시: 금색 대신 **국가 컬러** 파티클 (해당 국가 아레나)

**원칙 4: 성능 예산 (Performance Budget)**
- 목표: **60fps 유지** (200+ 엔티티 + 이펙트 상태), 프레임 타임 ≤ 16.6ms
- 동시 파티클 상한: 500개 (오브젝트 풀링 필수), 풀 크기 = 600 (여유 20%)
- 이펙트당 drawcall: 최대 3회 (메인 + 글로우 + 파티클)
- 자동 LOD: 화면 밖 이펙트는 렌더링 스킵, 30+ 엔티티 시 파티클 밀도 50% 감소
- Canvas 2D 최적화: `globalCompositeOperation = 'lighter'`로 발광 처리 (별도 패스 불필요)
- Gold 텍스트 풀: 동시 플로팅 텍스트 상한 20개 (초과 시 가장 오래된 것 즉시 제거)
- 프로파일링: 개발 중 `performance.now()` 기반 프레임 타임 모니터링 HUD (디버그 모드)

### 3.2 카테고리별 이펙트 비주얼 가이드

각 카테고리는 **고유한 시각 언어**를 가진다. 플레이어가 이펙트만 보고도 카테고리를 즉시 인식할 수 있어야 한다.

**STEEL (레드 계열) — 물리적/충격**
```
형태: 날카로운 슬래시 라인, 충격파 원, 파편
컬러: #EF4444(메인) → #FCA5A5(하이라이트) → #7F1D1D(그림자)
파티클: 금속 스파크, 먼지, 충격 잔해
특징: 히트 시 0.05초 프레임 정지(히트스탑) + 화면 미세 흔들림
```
- `whip` 전투 채찍: 호 형태의 붉은 슬래시 → 진화 시 전기 아크 추가
- `punch` 강철 주먹: 방사형 충격파 + 먼지 → 진화 시 대지 균열 이펙트
- `knife` 전투 단검: 직선 빠른 투사체 + 잔상 → 진화 시 진동 블러

**TERRITORY (블루 계열) — 에너지/플라즈마**
```
형태: 구체, 빔, 에너지 볼트, 전기 아크
컬러: #3B82F6(메인) → #93C5FD(하이라이트) → #1E3A8A(코어)
파티클: 에너지 입자, 전기 스파크, 잔상 트레일
특징: 투사체에 미세 발광(glow) + 히트 시 에너지 폭발
```
- `wand` 에너지 볼트: 원형 에너지탄 + 트레일 → 진화 시 유도 궤적
- `bow` 레일건: 직선 고속 빔 + 관통 잔상 → 진화 시 전자기 파동
- `axe` 전술 토마호크: 회전 투사체 + 바람 궤적 → 진화 시 유도 커브
- `shard` 클러스터탄: 분열 투사체 + 3방향 분기 → 진화 시 6방향

**ALLIANCE (퍼플 계열) — 연쇄/마법**
```
형태: 체인 라이트닝, 연결선, 파동 링, 오라
컬러: #8B5CF6(메인) → #C4B5FD(하이라이트) → #4C1D95(딥)
파티클: 마법 룬, 연결 스파크, 보라색 안개
특징: 적 사이 연결선 표시, 연쇄 타격 시 순차 플래시
```
- `lightning` 전술 번개: 적→적 체인 전기 → 진화 시 동시 체인 + 잔류전기
- `ping` 소나 펄스: 확산 원 + 벽 반사 → 진화 시 추적 파동
- `bridge` 냉각 폭탄: 얼음 결정 + 동결 오라 → 진화 시 서리 지대
- `fork` 분열탄: 분기 투사체 + 갈림길 표시 → 진화 시 다방향

**SOVEREIGNTY (그린 계열) — 방어/자연**
```
형태: 원형 실드, 육각 그리드, 지대 이펙트, 오라
컬러: #22C55E(메인) → #86EFAC(하이라이트) → #14532D(딥)
파티클: 육각형 조각, 에너지 장벽 파편, 치유 입자
특징: 지속 이펙트(오라, 지대)는 반투명 + 점멸
```
- `garlic` 방어 필드: 반투명 녹색 원 + 외곽 에너지 링 → 진화 시 파동 추가
- `bible` 가디언 위성: 궤도 도는 작은 구체 → 진화 시 공격 기능 추가
- `pool` 지뢰밭: 바닥 위험 표시(줄무늬) + 파열 → 진화 시 화염
- `stablecoin` 에너지 실드: 헥사곤 격자 실드 → 진화 시 반사

**MORALE (시안 계열) — 특수/에너지**
```
형태: 빔, 회전체, 폭격 마커, 핵폭발
컬러: #06B6D4(메인) → #67E8F9(하이라이트) → #164E63(딥)
파티클: 별, 에너지 파편, 충격파, 하강 궤적
특징: 대형 이펙트는 카메라 줌아웃 연출 동반
```
- `beam` 레이저 캐논: 두꺼운 직선 빔 + 가장자리 전기 → 진화 시 폭 증가
- `laser` 회전 레이저: 360도 라인 스윕 + 잔상 → 진화 시 이중 회전
- `airdrop` 공습: 낙하 궤적 + 지면 폭발 → 진화 시 융단 폭격
- `phishing` 궤도 폭격: 하늘→땅 빔 + 원형 폭발 → 진화 시 다중 빔
- `genesis` 전술 핵: 백색 플래시 + 버섯구름 + 충격파 → 궁극 시 블랙홀

### 3.3 이펙트-경제 연결

스킬 이펙트가 v36 토큰 이코노미의 **시각적 피드백 채널**이 된다.

**킬 이펙트 → Gold 시각화**:
| 킬 유형 | 이펙트 | Gold 보너스 | 시각 표현 |
|---------|--------|-----------|----------|
| 일반 킬 | 적 페이드아웃 | Base Gold | 작은 금색 코인 1~3개 |
| 크리티컬 킬 | 적 폭발 + 플래시 | Gold ×1.5 | 큰 코인 5~8개 + "CRITICAL" 텍스트 |
| 콤보 킬 (5+) | 적 연쇄 폭발 | Gold ×(1 + 콤보/10) | 코인 비 + 콤보 수 표시 |
| 진화 무기 킬 | 카테고리 컬러 폭발 | Gold ×1.25 | 카테고리 색상 코인 |
| 궁극 무기 킬 | 대형 폭발 + 카메라 쉐이크 | Gold ×2.0 | 보물상자 이펙트 |

**Gold 수집 이펙트**:
- 죽은 적에서 Gold 오브 생성 (경험치 젬과 구분 — 금색 육각형)
- 수집 시 플레이어 방향으로 자석 곡선 이동 + 도착 시 "+25G" 플로팅 텍스트
- `aggregator` (보급 체계) 패시브가 있으면 자석 반경 증가 + 자동수집

**Nation Score 이펙트** (v36 T2 연결):
- AI 에이전트 킬 시: 해당 에이전트의 팩션 엠블럼이 아래로 떨어지는 연출
- 에리어 확보 시: 확보된 영역이 잠시 해당 **국가 컬러**로 펄스 (점령 기여 시각화)
- Nation Score 마일스톤 도달: 화면 상단에 "NATION SCORE: 5,000" 골드 배너 (2초)

**전장 상점 구매 이펙트**:
- 구매 시: 골드 코인이 상점으로 빨려들어가는 애니메이션
- 아이템 장착: 해당 아이템 아이콘이 플레이어 위로 떠올라 흡수
- 높은 등급 구매: 금색 폭발 + 레어리티 컬러 테두리 플래시

## 4. UI/UX 전면 리디자인

### 4.1 현재 문제점 분석

현재 인게임 UI는 기능 추가 순서대로 누적되어 **통합적 디자인이 부재**하다.

**문제 1: 정보 과부하**
- MatrixHUD: HP, XP, 레벨, 킬수, 타이머, 무기슬롯 6개, 세이프존 — 한꺼번에 표시
- 플레이어가 전투 중 필요한 정보 vs 불필요한 정보의 구분이 없음
- 미니맵, 리더보드, 콤보 카운터까지 합치면 화면 10% 이상이 UI

**문제 2: 경제 정보 부재**
- v36 토큰 이코노미가 설계되었지만 인게임에서 Gold/Score를 **볼 수 없음**
- 레벨업 시 전투 스텟만 표시 — 경제적 가치(Gold 생산성)에 대한 정보 없음
- 매치 종료 후에야 보상을 확인 → 실시간 동기 부여 부재

**문제 3: 레벨업 모달의 UX 문제**
- 4개 선택지 중 경제형/전투형 구분이 안 됨 (모두 같은 카드 스타일)
- 자동 선택 로직이 전투력 기반 → 경제형 패시브를 추천하지 않음
- 모바일에서 카드가 작아 텍스트 잘림, 터치 영역 부족

**문제 4: 무기 슬롯 비효율**
- 6개 고정 슬롯이 화면 하단을 차지 — 빈 슬롯도 공간 사용
- 무기 레벨/진화 상태를 한눈에 파악 불가 (작은 숫자만)
- 패시브 스킬과 액티브 무기의 시각적 구분 없음

**문제 5: 디자인 비일관성**
- 일부 UI: Apex Tactical 스타일 (SK 팔레트, border-radius 0)
- 일부 UI: 이전 스타일 잔재 (둥근 모서리, 다른 폰트)
- 텍스트 크기/간격/컬러가 컴포넌트마다 다름

### 4.2 새로운 HUD 구조: "Tactical War Room"

전투 HUD를 **레이어 기반 정보 계층**으로 재설계한다.

**Layer 0: 항상 표시 (Always Visible)**
```
┌─────────────────────────────────────────────────────┐
│ [HP Bar]━━━━━━━━━━━━━━━━━━━━━━━━ [Gold: 2,450]      │  ← 좌상단
│ [XP Bar]━━━━━━━━━━━━━ Lv.7                          │
│                                                      │
│                    [GAME AREA]                        │
│                                                      │
│                                         [Timer 4:32] │  ← 우상단
│ [W1][W2][W3][P1][P2]                   [Kills: 47]  │  ← 좌하단/우하단
└─────────────────────────────────────────────────────┘
```

**Layer 1: 전투 핵심 (좌상단 클러스터)**
- HP 바: 너비 200px, 높이 8px, 카테고리 컬러 그라디언트
  - HP 감소 시 붉은 플래시 + 잔여 HP 바 서서히 따라감 (이중 바)
  - HP 20% 이하: 맥동 + 화면 가장자리 붉은 비네트
- XP 바: HP 바 아래, 너비 160px, 높이 4px, 금색
  - 레벨업 임박 시 발광 강화
- 레벨 배지: XP 바 우측, 원형 배지 "Lv.7"
- Gold 카운터: HP 바 우측, 금색 아이콘 + 숫자 (실시간 갱신)

**Layer 1: 전투 보조 (우상단 클러스터)**
- 타이머: Chakra Petch Bold, 큰 폰트, 1분 미만 시 레드
- 킬 카운터: 해골 아이콘 + 숫자, 킬 시 잠깐 스케일업
- 세이프존 경고: 존 밖일 때만 "DANGER ZONE" 깜빡임

**Layer 1: 무기 슬롯 (좌하단 — 가로 배열)**
- 액티브 무기: 아이콘 32px + 레벨 뱃지, 카테고리 컬러 테두리
- 패시브 스킬: 아이콘 24px + 레벨 뱃지, 뮤트 컬러 테두리 (작게)
- 진화 무기: 아이콘 테두리에 발광 + ★ 마크
- 빈 슬롯: 숨김 (보유 무기만 표시)
- 쿨다운: 아이콘 위 시계 방향 오버레이

**Layer 2: 상황별 표시 (Contextual — 기본 숨김, 트리거 시만 표시)**
- 콤보 카운터: 캐릭터 머리 위 (Canvas, 이미 구현) — 콤보 2+ 시만 표시
- 레벨업: 전체 화면 모달 — 레벨업 이벤트 시만
- 전장 상점: Tab키로 열기/닫기 — 명시적 조작 시만
- 리더보드: Tab 길게 누르기 — 명시적 조작 시만
- 미니맵: 우측 하단 소형 — 기본 OFF, M키 토글 (세이프존 축소 시 자동 ON)
- **원칙: Layer 0+1 이외의 정보는 화면에 상시 표시하지 않음** → 전투 시야 보장

**Layer 3: 경제 피드백 (오버레이 — 이벤트 시만)**
- Gold 획득: 화면 우측에서 좌로 슬라이드 "+150 GOLD" (1.5초)
- 콤보 보너스: 콤보 카운터 아래 "×2.5 BONUS" (1초)
- Nation Score: 화면 상단 중앙 배너 (2초)
- 마일스톤: "1,000 GOLD REACHED!" 축하 배너 (3초)

**디자인 토큰** (기존 SK 시스템 확장):
- 배경: rgba(9,9,11,0.85) — 반투명 블러
- 텍스트: #ECECEF (주), #A1A1AA (보조), #71717A (뮤트)
- 액센트: 카테고리별 컬러 (상기 6색)
- 골드: #F59E0B (Gold 관련 모든 요소)
- 보더: 1px solid rgba(255,255,255,0.08)
- 폰트: Chakra Petch (수치), Space Grotesk (라벨)

### 4.3 레벨업 모달 리디자인

레벨업 선택을 **전투/경제 이중 차원의 전략적 의사결정**으로 전환한다.

**현재 레벨업 모달**: 4개 스킬 카드 (이름 + 스텟 변화 + 카테고리 색상)
**새 레벨업 모달**: 4개 전략 카드 (전투 효과 + 경제 효과 + 추천 태그)

**카드 레이아웃**:

## 5. 스킬 아이콘 전면 리디자인

### 5.1 아이콘 디자인 시스템

픽셀아트 PNG를 **통일된 SVG 플랫 아이콘 시스템**으로 교체한다.

**디자인 원칙**:
1. **형태 즉시 인지**: 16px에서도 무기가 뭔지 알 수 있어야 함
2. **카테고리 색상**: 메인 컬러 = 카테고리 컬러 (STEEL=레드, TERRITORY=블루 등)
3. **레벨별 시각 변화**: 기본→진화→궁극에 따라 디테일 추가
4. **일관된 그리드**: 모든 아이콘이 24×24 기본 그리드에 정렬

**아이콘 사양**:
```
포맷: SVG (인라인 React 컴포넌트)
기본 크기: 24×24 viewBox
실사용 크기: 16px (HUD 슬롯), 32px (레벨업 카드), 48px (디버그 패널)
스트로크: 2px (기본), 1.5px (작은 사이즈)
필: 카테고리 메인 컬러, opacity 조절로 깊이감
배경: 투명 (상위 컨테이너가 배경 제공)
```

**상태별 아이콘 변형**:
| 상태 | 시각 처리 |
|------|----------|
| 미보유 | 그레이스케일 (#71717A), opacity 0.5 |
| 보유 Lv1~10 | 카테고리 컬러, 기본 형태 |
| 진화 Lv11+ | 카테고리 컬러 + **빛줄기 방사선** 추가 + 테두리 glow |
| 궁극 Lv20 | 카테고리 컬러 + **금색 크라운** 오버레이 + 풀 glow |
| 쿨다운 중 | 반시계 방향 sweep 오버레이 (어두운 반원) |
| 사용 불가 | 빨간 X 오버레이 |

**구현 방식**: React 컴포넌트 (`SkillIconSVG.tsx`)
```typescript
// 기존: import SkillIcon (PNG 기반)
// 새로: import SkillIconSVG (SVG 인라인, 상태 반응형)
<SkillIconSVG
  type="whip"
  size={32}
  level={11}           // → 진화 상태 자동 판단
  cooldownPercent={0.3} // → sweep 오버레이
  disabled={false}
/>
```

**폴백 전략**: SVG 로딩 실패 시 → lucide-react의 유사 아이콘으로 자동 폴백 (Zap, Shield, Sword 등)

## 6. 사용자가 놓치기 쉬운 확장 설계

사용자가 놓치기 쉬운 **핵심 확장 설계** 영역. 전투-경제 통합의 깊이를 더한다.

### 6.1 사운드 디자인 리매핑

현재 코딩 밈 사운드를 군사/전략 사운드로 전환:
| 이벤트 | 현재 | 새 사운드 | 경제 추가 |
|--------|------|----------|----------|
| 무기 발사 | 타자기/키보드 | 무기별 고유 사운드 (총, 레이저, 폭발) | — |
| 적 처치 | 팝/클릭 | 타격감 있는 폭발음 | Gold 코인 소리 (차잉) |
| 레벨업 | 알림음 | 군사 팡파레 | — |
| 진화 | 없음 | 에너지 차지 → 폭발 (2단계) | 보너스 Gold 소리 |
| Gold 수집 | 없음 ★ | 코인 수집음 (띠링) | — |
| 상점 구매 | 없음 ★ | 금고 열림 → 코인 지불 | — |
| 콤보 도달 | 없음 ★ | 단계별 징글 (5콤보/10콤보/20콤보) | — |

★ = 완전 신규 사운드 이벤트

### 6.2 킬피드 시스템

화면 우측에 실시간 킬피드 표시 (FPS 게임 스타일):
```
[NEXUS] Neo ×─── 전투 채찍 ───× Trinity [VIPER]    ← 팩션 간 PvP
[NEXUS] Morpheus ×─── 레일건 ───× [Bot Lv.12]      ← 몬스터 킬
```
- 팩션 태그 [TAG]는 팩션 컬러로 표시 (FactionSummary.color 사용)
- 에이전트 이름은 PlayerClass 기반 (Neo, Trinity, Morpheus 등)
- 국가 아레나 이름은 HUD 상단에 별도 표시 (예: "🇰🇷 KOREA ARENA")
- 최근 5건 표시, 3초 후 페이드아웃
- 자신의 킬: 금색 하이라이트 + "+25G" 옆에 표시
- 자신이 죽음: 빨간색 하이라이트

### 6.4 AI 에이전트 개성 강화

9개 AI 에이전트는 **PlayerClass 기반** (`arena-agents.config.ts`)이며, 각 에이전트에 **전투 성향** + **경제 성향**을 부여한다.
에이전트는 **팩션 소속**으로 표시되며 (매치마다 팩션 랜덤 할당), **국가 아레나**에서 Nation Score를 놓고 경쟁한다.

| 에이전트 (Class) | 칭호 | 성격 | 전투 성향 | 경제 성향 | 선호 무기 |
|-----------------|------|------|----------|----------|----------|
| **Neo** | The One | balanced | 균형 (STEEL+TERRITORY) | 상황적 | 전투 단검 + 에너지 볼트 |
| **Trinity** | The Hacker | aggressive | 화력 (TERRITORY) | 고소비 (상점 적극) | 레일건 + 클러스터탄 |
| **Morpheus** | The Captain | balanced | 지휘 (ALLIANCE) | 투자형 | 전술 번개 + 연합 전선 |
| **Tank** | The Operator | defensive | 방어 (SOVEREIGNTY) | 축적형 | 방어 필드 + 에너지 실드 |
| **Cypher** | The Traitor | collector | 수집 (INTELLIGENCE) | 고축적 (Gold 모음) | 보급 체계 + 전리품 |
| **Niobe** | The Pilot | aggressive | 공격 (MORALE) | 고소비 | 레이저 캐논 + 공습 |
| **Oracle** | The Seer | defensive | 정보 (INTELLIGENCE) | 장기투자 | 정보 분석 + 국가 충성 |
| **Mouse** | The Programmer | collector | 효율 (ALLIANCE) | 효율투자 | 냉각 폭탄 + 소나 펄스 |
| **Dozer** | The Engineer | defensive | 진지 (SOVEREIGNTY) | 균형 | 지뢰밭 + 가디언 위성 |

**경제 행동 AI**:
- `collector` 성격: Gold 오브를 우선 수집, 상점에서 경제 아이템 구매
- `aggressive` 성격: 킬 추구, 상점에서 공격력 아이템 구매
- `defensive` 성격: 생존 우선, 상점에서 HP 키트/실드 구매
- `balanced` 성격: 상황에 따라 전투/경제 전환

### 6.5 매치 진행 페이즈 시스템

5분 매치를 3개 페이즈로 분할하여 경제/전투 리듬 설계:

| 페이즈 | 시간 | 지속 | 특성 | Gold 배율 | Score 배율 |
|--------|------|------|------|----------|-----------|
| **전초전** | 0:00~1:30 | 90초 | 약한 적, 레벨업 집중 | ×1.0 | ×1.0 |
| **교전기** | 1:30~3:30 | 120초 | 세이프존 축소 시작, 중~강 적 | ×1.5 | ×1.5 |
| **결전** | 3:30~5:00 | 90초 | 최소 세이프존, 엘리트 스폰 | ×2.5 | ×3.0 |

총 300초(5분) = arena.config의 `ROUND_DURATION` 일치. 세이프존 축소는 교전기 시작(90초)부터 시작.

- 페이즈 전환 시: 화면 전체 플래시 + "PHASE 2: ENGAGEMENT" 배너
- 결전 시작: 배경 음악 BPM 증가 + 카메라 약간 줌아웃
- 경제형 빌드는 결전 페이즈에서 극대화 (후반 Gold/Score 폭등)

## 8. 기술 방향

### 8.1 렌더링 유지: Canvas 2D

기존 Canvas 2D 렌더링 파이프라인을 유지한다. 이유:
- 78개 렌더링 모듈이 이미 Canvas 2D 기반으로 최적화됨
- 수백 엔티티 스프라이트 렌더링에 Canvas 2D가 WebGL보다 가벼움
- Three.js/PixiJS 전환 비용 >> 비주얼 개선 효과

**이펙트 강화 방법** (Canvas 2D 내에서):
- `globalCompositeOperation = 'lighter'`: 발광/블렌딩 효과
- `shadowBlur` + `shadowColor`: 글로우 효과
- `createRadialGradient()`: 에너지/오라 표현
- Canvas offscreen 버퍼: 복잡한 이펙트 프리렌더링
- `Path2D`: SVG 아이콘 캔버스 렌더링 (HUD 캔버스 레이어)

### 8.2 아이콘 기술: 인라인 SVG React 컴포넌트

```
현재: SkillIcon.tsx → <img src="/assets/skills/{cat}/{type}.png">
변경: SkillIconSVG.tsx → <svg viewBox="0 0 24 24">...</svg>
```

**장점**:
- 번들 크기: SVG 인라인 < PNG 30장 (네트워크 요청 0)
- 동적 컬러: props로 실시간 색상 변경 (카테고리, 상태)
- 크기 무한 확장: 16px~128px 선명
- 애니메이션: CSS/JS로 회전, 펄스, 페이드 가능

**구현**: 컴포넌트 내부에 `switch(type)` + `<svg>` JSX 직접 반환
또는 아이콘 Map 객체로 SVG path data를 관리

### 8.3 경제 시스템 아키텍처

```
[Combat System] → onKill(enemy, weapon) → [Economy Manager]
                                              ├→ addGold(amount, source, modifiers)
                                              ├→ addScore(amount, source)
                                              └→ triggerVisualFeedback(type, position)

[Economy Manager]
  ├── goldState: { current, total, perMinute, history[] }
  ├── scoreState: { current, total, perMinute }
  ├── modifiers: { baseMultiplier, passiveBonus, comboMultiplier, phaseMultiplier }
  └── shop: { items[], purchaseHistory[], inflationFactor }
```

- `EconomyManager` 클래스: 순수 로직 (Canvas/React 무관)
- `useEconomy` 훅: React 상태 연결 (HUD/상점 컴포넌트용)
- `economyCanvas.ts`: Canvas 이펙트 렌더링 (코인 파티클, 플로팅 텍스트)

### 8.4 데이터 호환성

무기 키(key)를 변경하지 않으므로 **기존 시스템과 100% 호환**:
- `WEAPON_DATA[type]`: `.name`, `.desc`, `.color` 필드만 교체
- `fireWeapon(type)` 디스패처: 변경 없음
- `progressive-tree.config.ts`: 카테고리 매핑 변경 (코드→군사 이름)
- 렌더러 파일: 이펙트 비주얼만 교체, 로직 유지

### 8.5 파일 변경 범위 예측

| 변경 유형 | 파일 수 | 설명 |
|----------|---------|------|
