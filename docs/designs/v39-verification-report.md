## 🚨 Critical Issues (즉시 수정 필요)

### C-01: 에폭 타이밍 호환성 — 15분 라운드가 기존 10m15s 에폭과 충돌
- **위치**: §3 (게임 사이클), §9.1 (에폭 시스템 확장)
- **근거**: v33 에폭 페이즈 타이밍은 **컴파일 타임 상수**로 정의됨 (v33 §7.1). 20Hz 기반 절대 틱 오프셋(5,800/200/3,600/2,400/100/200 ticks)이 하드코딩. 런타임 변경 불가 — 서버 재빌드 필수.
- **문제**: v39 §9.1에서 "기존 EpochManager의 인터페이스는 유지하되, 내부 타이밍과 페이즈 정의만 변경"이라고 적었으나, 이것은 **완전 새로운 RoundEngine**을 만드는 것이지 기존 EpochManager "수정"이 아님. 마이그레이션 전략이 불명확.
- **영향**: 기존 10m15s 에폭을 사용하는 코드가 서버 전체에 퍼져 있음 (DominationEngine 6에폭 주기, CapturePointSystem, ScoreAggregator 등)
- **권장 수정**:
  1. EpochManager를 **폐기(deprecate)**하고 새 RoundEngine을 만든다고 명시
  2. 하위 호환 기간: EpochManager 인터페이스를 RoundEngine이 구현하되, 새 타이밍 사용
  3. DominationEngine의 "6에폭(=1시간) 평가" → "4라운드(=1시간) 평가"로 변환표 추가

### C-02: Sovereignty 모델 불일치 — v39 일일 정산 vs v35 연속 지배 기반
- **위치**: §5.2 (일일 지배 판정), §5.3 (국가 주권)
- **근거**: v35 §12.3에 정의된 Sovereignty Escalation Ladder:
  - None → (6에폭 연속 1시간 지배) → Active Domination
  - Active Domination → (24시간 연속) → Sovereignty
  - Sovereignty → (7일 연속) → Hegemony
- **문제**: v39는 "일일 정산(UTC 00:00)에 최다 RP 팩션이 지배"라는 **점수 기반** 모델을 사용. 기존 v35는 "연속 지배 시간" 기반 모델. 두 모델이 **근본적으로 다름**.
- **영향**: v35의 Sovereignty→Hegemony 에스컬레이션 래더가 v39에서 어떻게 적용되는지 불명확. Hegemony 보상(100 $AWW + 10/멤버)의 달성 조건도 미정의.
- **권장 수정**:
  1. v39의 지배 모델이 v35를 **대체**(replace)하는지 **공존**(coexist)하는지 명시
  2. Sovereignty Escalation Ladder를 v39 모델로 재정의:
     - 일일 정산 1회 지배 확정 → Active Domination
     - 3일 연속 일일 정산 지배 → Sovereignty
     - 14일 연속 → Hegemony
  3. §5.4의 연속 지배 스트릭 보너스를 Hegemony 래더와 통합

### C-03: 국가 티어별 지역 수와 아레나 크기 충돌
- **위치**: §2.1 (티어별 지역 수)
- **근거**: v35 §13.1에서 국가 티어별 아레나 크기가 정의됨:
  - S: 6,000px, A: 4,500px, B: 3,500px, C: 2,500px, D: 1,500px
- **문제**: v39는 모든 지역 아레나를 **8,000×8,000px**로 통일 (§2.3 arena 필드). 그러나:
  1. D티어 국가(기존 1,500px 아레나)의 3개 지역이 각각 8,000px = 기존 대비 5.3배 확대
  2. S티어 국가(기존 6,000px)의 7개 지역이 각각 8,000px = 총 면적 7×64M = 448M px² vs 기존 36M px²
  3. 서버 부하: 한 국가에 7개 × 30명 = 210명 동시 접속 가능 (기존 50명 상한)
- **영향**: 소국 지역이 너무 넓어 파밍/전투 밀도가 극히 낮아짐. 대국은 서버 부하 7배 증가.
- **권장 수정**:
  1. 지역 아레나 크기를 **국가 티어에 비례**하여 차등화:
     - S: 6,000px/지역, A: 5,000px, B: 4,000px, C: 3,000px, D: 2,500px
  2. 또는 지역 아레나 크기를 MaxPlayers에 비례:
     - 10명 → 4,000px, 20명 → 6,000px, 30명 → 8,000px
  3. D티어 국가의 지역 수를 2~3에서 1~2로 축소 검토 (§10 리스크에서도 언급)

### C-04: S티어 국가 수 불일치 — v39 ~10 vs v35 8개
- **위치**: §2.1 (티어별 지역 수)
- **근거**: v35 §13.1 정의: S티어 = 8개국 (미국, 중국, 러시아, 인도, 일본, 독일, 영국, 프랑스)
- **문제**: v39 §2.1에서 S티어 "~10"으로 기술. 기존 확정된 8과 불일치. 추가 2개국이 어디인지 미정의.
- **영향**: 지역 데이터 생성 시 혼란 (Phase 1 작업)
- **권장 수정**: v35 기준 8개국으로 확정하거나, 추가할 2개국(브라질, 한국?)을 명시

### C-05: 캡처 포인트 시스템 누락
- **위치**: §3 전체 (게임 사이클)
- **근거**: v33/v35에서 아레나당 3개 캡처 포인트(Resource/Buff/Healing, 80px 반경, 5초 캡처)가 중요 전투 요소
- **문제**: v39 기획서에서 캡처 포인트 시스템이 **전혀 언급되지 않음**. PvE 중 캡처 포인트 운영 여부, 배틀로얄에서 캡처 포인트가 전략적 거점으로 활용되는지 미정의.
- **영향**: 기존 CapturePointSystem 코드와의 호환, 전략적 깊이 손실
- **권장 수정**:
  1. PvE 페이즈: 캡처 포인트 활성 (자원 채취 + 버프 확보 기회)
  2. 배틀로얄 페이즈: 캡처 포인트가 세이프존 힌트 또는 에어드롭 위치로 재활용
  3. 또는 캡처 포인트를 자원 노드로 대체한다고 명시적으로 폐기

### C-06: 전쟁 시스템(War System)과 영토 지배의 관계 미정의
- **위치**: §4, §5 전체
- **근거**: v35 §9에 정의된 War State Machine (선포→준비(24h)→활전(72h)→종료→쿨다운(24h))
- **문제**: v39는 배틀로얄로 자연스럽게 팩션 간 PvP가 이루어지는데, 기존 전쟁 선포 시스템과 어떻게 공존하는지 미정의.
  - 전쟁 선포 없이도 배틀로얄에서 PvP가 되는가? → 그렇다면 전쟁 선포의 의미는?
  - 전쟁 선포 시 추가 효과(Cross-Arena Invasion, 방어 보너스)가 배틀로얄에 적용되는가?
- **권장 수정**:
  1. "배틀로얄 PvP"와 "전쟁 선포 PvP"를 구분하여 정의
  2. 배틀로얄 = 자동 PvP (매 라운드, 별도 선포 불필요)
  3. 전쟁 선포 = Cross-Arena Invasion (다른 국가 지역 원정) + 추가 보너스/페널티
  4. §7.6 연맹 전쟁과의 관계도 명확화

- **근거**: v37 §6.6에 9개 필드 상점 아이템이 정의됨 (HP Kit, XP Boost, Shield, DMG+5%, SPD+5% 등)
- **문제**: v39에서 매 라운드 레벨/스킬이 리셋되는데, Gold은 누적된다고 기술 (§3.3). 그렇다면 필드 상점 아이템도 매 라운드 리셋? Gold 누적이면 후반 라운드에 상점 아이템 과다 구매로 밸런스 붕괴 가능.
- **권장 수정**:

### H-04: Match Phase(v37)와 Round Phase(v39)의 관계 불명확
- **위치**: §3.1, §3.2
- **근거**: v37 §6.5에 5분 Match Phase 정의 (Skirmish 90s → Engagement 120s → Final Battle 90s). 이것은 현재 에폭 내부의 PvP 구간 sub-phase.
- **문제**: v39의 5분 배틀로얄 페이즈에 v37의 Match Phase를 적용하는지 불명확. 적용한다면:
  - BR 0~1:30 = Skirmish (x1.0 Gold/Score)
  - BR 1:30~3:30 = Engagement (x1.5, 세이프존 1단계 수축)
  - BR 3:30~5:00 = Final Battle (x2.5 Gold, x3.0 Score)
  이 배율이 적용되면 게임 경험이 크게 달라짐.
- **권장 수정**: BR 페이즈 내 Match Phase sub-timing을 명시적으로 정의하거나 폐기 선언

### M-03: 96라운드/일 계산 오류
- **위치**: §5.1
- **문제**: "하루 96라운드 (15분 × 96 = 24시간)"이지만, §3.3에서 정산(15초) + 대기(10초) = 25초 추가. 실제 1라운드 = 15분 25초 = 925초. 24h / 925s = **93.4라운드/일**. 또한 서버 재시작, 유지보수 시간을 고려하면 더 적음.
- **권장**: ~93라운드/일로 수정, 또는 정산 시간을 라운드에 포함시켜 "15분 라운드 (PvE 9:45 + BR 5:00 + Settlement 0:15 = 15:00)"로 조정

### M-04: 무소속 플레이어(solo-faction) 처우
- **위치**: §4.1
- **문제**: 무소속 = 1인 팩션으로 자동 생성. 배틀로얄에서 1인 팩션은 Underdog Boost(HP +30%, DMG +20%) + NPC 2명 지원을 받지만, 5인 팩션 대비 여전히 불리. 무소속 플레이어 경험이 "팩션에 가입하지 않으면 게임이 안 됨" 수준이면 신규 유저 이탈.
- **권장**: 신규 유저를 위한 "자동 팩션 배정" 시스템 확장 (§10 리스크에서 언급했으나 상세 미정의). 또는 무소속 전용 보너스 (PvE 자원 ×2, 생존 시 RP ×2 등)

### M-05: 지역 건물 인수 시 밸런스
- **위치**: §7.8
- **문제**: "건물은 지배 상실 시 파괴되지 않음 (새 지배 팩션이 인수)". 이것은 선점 팩션에게 극도로 유리 — 초반에 건물 3개 건설 후 지배를 뺏겨도 적이 건물 혜택을 받음. 탈환 시 다시 내 건물이 됨.
- **권장**: 지배 상실 시 건물 **중립화**(효과 비활성) → 새 지배 팩션이 Gold으로 **인수**(활성화 비용 50%) 또는 파괴 선택

### M-06: 팩션 인원/지역 상한 수식의 엣지 케이스
- **위치**: §7.2
- **문제**: `max(10, total_members / regions_count)`. 50명 팩션이 한 국가(5지역)에 들어가면 10명/지역. 그런데 팩션원이 10명 이하인 팩션은 모든 멤버가 같은 지역에 들어가도 상한 10명 이내. → 소규모 팩션은 집중, 대규모 팩션은 분산 강제. 의도된 밸런스인지, regions_count가 "해당 국가 지역 수"인지 "전체 게임 지역 수"인지 불명확.
- **권장**: `regions_count` = "해당 국가의 지역 수"로 명시

### M-07: 토큰 보상 일일 상한과 영토 보상 합산
- **위치**: §5.4, §9.2
- **근거**: v35에서 일일 토큰 상한 = 5,000 토큰/플레이어/일
- **문제**: v39에서 영토 보상(지역 50토큰/일 + 국가 200 $AWW/일)이 개인 보상에 추가됨. 한 팩션이 10개 지역을 지배하면 멤버 1인당 500 토큰/일 추가. 이것이 기존 5,000 상한에 포함되는지, 별도인지 불명확.
- **권장**: 영토 보상은 **팩션 국고에 입금** (개인 상한과 분리), 개인에게는 팩션이 분배하는 구조로 정리

### M-08: KillValidator와 팩션 면역의 상호작용
- **위치**: §4.2
- **근거**: v33 KillValidator — 서버가 kill_report를 검증 (거리, 쿨다운, 데미지, 상태)
- **문제**: 팩션 면역 검사가 KillValidator 파이프라인에 추가되어야 함. 클라이언트에서 아군 투사체가 관통하더라도, 서버에서도 동일하게 "아군 kill_report 거부" 처리 필요.
- **권장**: KillValidator에 Step 0 (faction check) 추가: `if killer.factionId == target.factionId → reject` — 기술 아키텍처 §8.1에 명시

### M-09: Reckoning Era(4주차) 시즌 종료와 영토 리셋 충돌
- **위치**: §7.1
- **근거**: v35 §13.2 — Reckoning Era: "3분 배틀 사이클, 최종 러시"
- **문제**: v39 §7.1에서 Reckoning = "모든 지배 연속 보너스 리셋, 최종 쟁탈전"이라 했으나, v35의 "3분 배틀 사이클"과 v39의 "15분 라운드"가 충돌. Reckoning 때만 특별 라운드 (7.5분 = PvE 5분 + BR 2.5분?)를 운영하는지?
- **권장**: Reckoning Era 특별 규칙을 상세 정의 (라운드 시간 단축, RP 배율 증가, 최종 순위 결정 기준)
