## 🚨 Critical Issues (즉시 수정 필요)

### C-01: 자원 종류 불일치 (§5 vs §7)
- **위치**: §5 경제 시스템 (5종) vs §7 국가 데이터 (6종)
- **증거**: §5에서 "Energy, Metal, Food, Tech, Gold" 5종 / §7 CountryData에 `resources: { [key: string]: number }` 6종 언급
- **영향**: 자원 생산/소비/무역 로직 전반에 혼란
- **수정안**: 6종으로 통일 (Energy, Metal, Food, Tech, Gold, **Manpower**) 또는 5종 확정 후 전체 업데이트

### C-02: 거버넌스 권한 충돌 (§5 vs §15)
- **위치**: §5 "팩션 리더가 경제 정책 결정" vs §15 "토큰 홀더가 쿼드라틱 보팅으로 결정"
- **증거**: §5.5에서 `faction_leader` → 세율/무역정책 결정 / §15.4에서 `sqrt(tokens)` = 투표 가중치
- **영향**: 동일 정책에 대해 2개의 결정 주체가 충돌 → 게임 메카닉 자체가 모순
- **수정안**: **이원 거버넌스** 명시 — 팩션 리더(군사/외교 결정) + 토큰 홀더(경제 정책 결정) 역할 분리

### C-03: Sovereignty 용어 불일치
- **위치**: §3 "sovereignty" vs §4 "control_level" vs §7 "sovereignty_index"
- **증거**: 동일 개념에 3가지 다른 이름 사용
- **영향**: 구현 시 혼란, API 스키마 불일치
- **수정안**: `sovereignty_level` (0-100 float)로 통일

### C-04: 스마트 컨트랙트 감사 부재 (§15)
- **위치**: §15 블록체인 전체
- **증거**: 195개 토큰 + Treasury + Oracle + Governance 컨트랙트 — 감사(audit) 계획 전무
- **영향**: DeFi 해킹으로 전체 토큰이코노미 붕괴 가능 (2025년 기준 DeFi 해킹 $2B+/년)
- **수정안**: Phase 10에 "S46.5: 보안 감사" Step 추가, Testnet 배포 → 감사 → Mainnet 순서

### C-05: Oracle 조작 취약점 (§15.6)
- **위치**: DefenseOracle.sol
- **증거**: 5분 TWAP만으로 방어 버프 결정 — 낮은 유동성 토큰에서 $1K으로 시총 10x 조작 가능
- **영향**: Siege 전투 직전 자전거래로 방어 버프 극대화 → 공정성 파괴
- **수정안**: (a) 최소 유동성 $10K 이상일 때만 버프 적용, (b) TWAP 기간 1시간으로 연장, (c) 급격한 변동(>50%/1hr) 시 버프 동결

## 🪙 토큰이코노미 지속가능성

### 핵심 리스크: Death Spiral
```
GDP↓ → 바이백↓ → 토큰가격↓ → 방어↓ → 정복 → GDP 더↓ → 완전 붕괴
```
- **판정**: ⚠️ 현재 설계는 Death Spiral 취약
- **필수 수정**: 최소 바이백 플로어 + 구제금융 메카닉 + 아레나 밸런싱

### 유동성 분산 문제
- 195개 독립 풀 = 극도로 분산된 유동성 → 대부분 토큰이 비활성화
- **판정**: 🚨 현실적으로 지속불가
- **필수 수정**: 단계적 토큰 출시 (S등급 8국 → A등급 20국 → ...) 또는 "리전 토큰" (대륙별 6개)

### Pay-to-Win 우려
- §15에서 10만 토큰 이상 보유 → "+30% 방어 버프"
- **판정**: ⚠️ P2W 인식 불가피
- **수정안**: 방어 버프 상한을 +15%로 하향 + 스킬 기반 오버라이드 명시

### 증권법 리스크
- "토큰 구매 → 인게임 효과 → 가치 상승 기대" 구조 = Howey Test 위반 가능
- **판정**: ⚠️ 법적 검토 필수
- **수정안**: (a) 유틸리티 토큰 면책 조항 강화, (b) 법률 자문 확보, (c) 규제 우호 관할권 법인 설립
