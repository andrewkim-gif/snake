# PLAN: v39 — 팩션 기반 영토 지배 시스템 (Faction Territory Dominance)

> v39 — 국가별 지역(Region) 분할 + 팩션 기반 배틀로얄 + 영토 지배 시스템
> 기존 v33 온라인 Matrix + v35 토큰 이코노미 + v37 인게임 시스템을 확장
> 작성일: 2026-03-11

---

## 1. 개요

### 1.1 핵심 비전

현재 AI World War의 Matrix 전투는 **개인 기반 배틀**이다. 유저가 국가 아레나에 진입하면 10분15초 에폭 동안 PvE+PvP를 반복하고, 개인 Nation Score가 집계되어 토큰을 받는다.

**v39는 이 구조를 근본적으로 바꾼다**:
- 국가를 **여러 지역(Region)**으로 분할하여 각 지역이 독립 아레나가 됨
- 10분 PvE 파밍 → 5분 **팩션 기반 배틀로얄** → 15분 라운드 사이클
- 배틀로얄에서 **같은 팩션은 공격 불가** (팩션 = 팀)
- 최후까지 살아남은 팩션이 **해당 지역의 지배 포인트** 획득
- **1일 동안 가장 많은 라운드에서 생존한 팩션**이 해당 지역을 지배
- 한 국가의 **모든 지역을 지배한 팩션**이 해당 국가의 **주권(Sovereignty)** 획득

이것은 단순 PvP 아레나를 넘어, **전략적 영토 쟁탈전**으로 게임을 진화시킨다.

### 1.2 현재 시스템 vs 신규 시스템

| 항목 | 현재 (v33~v37) | 신규 (v39) |
|------|---------------|------------|
| **아레나 단위** | 국가 1개 = 아레나 1개 | 국가 1개 = 3~7개 지역 아레나 |
| **사이클** | 10분15초 에폭 (Peace+War+Shrink) | **15분 라운드** (10분 PvE + 5분 배틀로얄) |
| **PvP 규칙** | 개인 vs 개인 (War 페이즈) | **팩션 vs 팩션** (같은 팩션 공격 불가) |
| **점수 단위** | 개인 Nation Score | 팩션 생존 포인트 + 개인 기여 점수 |
| **영토 개념** | 국가 단위 주권 (24시간 지배) | **지역 단위 지배** → 국가 통일 = 주권 |
| **파밍** | PvE 킬 + XP 젬 (범용) | **국가별 고유 자원** 채취 + 팩션 경제 투입 |
| **보상** | 에폭당 토큰 지급 | 라운드 생존 상금 + 지역 지배 보너스 + 국가 주권 보너스 |

### 1.3 핵심 변경 포인트

1. **국가 → 지역 분할**: 195개국 각각을 3~7개 대표 지역으로 분할 (S티어 7지역, A 5, B 4, C 3, D 3)
2. **15분 라운드 사이클**: 기존 10분15초 에폭 → 10분 PvE + 5분 배틀로얄
3. **팩션 팀전**: 배틀로얄에서 동일 팩션 멤버 간 공격 면역 (팩션 = 팀)
4. **지역 지배 메커니즘**: 라운드 생존 → 일일 누적 → 지역 지배 → 국가 주권
5. **국가별 고유 자원**: 한국=반도체, 미국=석유, 일본=자동차 등 국가 특색 자원
6. **팩션 자원 경제**: 채취한 자원 → 팩션 국고 → 전쟁/외교/인프라 비용
7. **오프라인 방어**: 24시간 지배 유지를 위한 수비 메커니즘 (NPC 수비대, 동맹 지원)

## 2. 지역(Region) 시스템

### 2.1 국가 → 지역 분할 구조

각 국가는 **국가 티어에 따라 3~7개 대표 지역**으로 분할된다. 지역은 실제 지명에 기반하며, 각각 독립 아레나로 기능한다.

```
┌──────────────────────────────────────────────────────────────────┐
│ COUNTRY (국가)                                                    │
│                                                                    │
│ ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐          │
│ │ Region 1 │  │ Region 2 │  │ Region 3 │  │ Region N │          │
│ │ "수도권"  │  │ "산업벨트"│  │ "항구도시"│  │ "자원지대"│          │
│ │ Arena    │  │ Arena    │  │ Arena    │  │ Arena    │          │
│ │ Max 30   │  │ Max 30   │  │ Max 30   │  │ Max 30   │          │
│ └──────────┘  └──────────┘  └──────────┘  └──────────┘          │
│                                                                    │
│ 모든 지역 지배 = 국가 주권 (Sovereignty) 획득                       │
└──────────────────────────────────────────────────────────────────┘
```

**티어별 지역 수**:

| 국가 티어 | 지역 수 | 예시 | 국가 수 |
|----------|---------|------|---------|
| **S** (강대국) | 7 | 미국, 중국, 러시아, 인도, 일본, 독일, 영국, 프랑스 | 8 |
| **A** (지역 강국) | 5 | 한국, 브라질, 캐나다, 호주, 이탈리아, 터키, 사우디 | 20 |
| **B** (중견국) | 4 | 태국, 폴란드, 이집트, 멕시코 | 40 |
| **C** (소국) | 3 | 싱가포르, 포르투갈, 체코 | 80 |
| **D** (도시국가/소도서국) | 3 | 모나코, 바티칸, 투발루 | 47 |

**예시 — 한국 (A티어, 5지역)**:
| 지역 | 실제 기반 | 고유 자원 | 환경 테마 | 지역 효과 |
|------|----------|----------|----------|----------|
| 서울 수도권 | Seoul | 테크 | 도시 스카이라인 | PvP 데미지 +10% (밀집) |
| 경기 산업벨트 | Gyeonggi | 반도체 | 산업단지 | 생산 속도 +15% |
| 부산 항구 | Busan | 무역 물자 | 항구/해안 | 교역 수수료 -20% |
| 제주 자원지대 | Jeju | 식량 | 화산섬/밀림 | HP 재생 +10% |
| DMZ 전선 | Paju | 군수품 | 전쟁 폐허 | 방어 +15% (요새 지형) |

### 2.2 지역 속성 (국가별 고유 자원/환경)

**7가지 지역 유형** (각 국가의 지역은 아래 유형 중 배정):

| 유형 | 고유 자원 | 맵 환경 | 특수 효과 | PvE 특징 |
|------|----------|---------|----------|---------|
| **수도 (Capital)** | Tech | 고층 빌딩, 네온 | PvP 데미지 +10% | 엘리트 몹 다수 |
| **산업 (Industrial)** | Minerals | 공장, 컨베이어벨트 | 생산 속도 +15% | 기계형 몹 |
| **항구 (Port)** | Gold (교역) | 항만, 크레인 | 교역 수수료 -20% | 해양 몹 |
| **농업 (Agricultural)** | Food | 들판, 농장 | HP 재생 +10% | 야생 몹 |
| **군사 (Military)** | Oil (군수) | 기지, 벙커 | 방어 +15% | 군사 로봇 몹 |
| **자원 (Resource)** | Minerals+Oil | 광산, 유전 | 자원 수집량 ×1.5 | 자원 수호자 몹 |
| **문화 (Cultural)** | Influence | 유적, 사원 | 외교 영향력 +20% | 문화 수호자 몹 |

**지역 환경은 맵 비주얼에 직접 반영**:
- 산업 지역 → 철골 구조물, 기어 회전, 연기 파티클
- 농업 지역 → 녹색 들판, 나무, 강줄기
- 군사 지역 → 참호, 바리케이드, 지뢰밭

### 2.3 지역 데이터 모델

```typescript
interface Region {
  id: string;                    // "KOR_seoul", "KOR_busan" 등
  countryCode: string;           // "KOR"
  name: string;                  // "서울 수도권"
  nameEN: string;                // "Seoul Capital"
  type: RegionType;              // capital | industrial | port | agricultural | military | resource | cultural
  primaryResource: ResourceType; // tech | minerals | gold | food | oil | influence
  environmentTheme: string;      // "urban_skyline" | "industrial_zone" | ...

  // 지역 효과 (PvE/PvP 버프)
  effects: {
    pvpDamageMultiplier: number;   // 1.1 = +10%
    pveSpawnRate: number;          // 1.0 = 기본
    resourceGatherRate: number;    // 1.5 = 수집량 50% 증가
    hpRegenBonus: number;          // 0.1 = +10%
    defenseBonus: number;          // 0.15 = +15%
  };

  // 지배 상태
  dominance: {
    controllingFactionId: string | null;  // 현재 지배 팩션
    controlSince: Date | null;            // 지배 시작 시점
    dailyScores: Record<string, number>;  // factionId → 일일 생존 포인트 합산
    lastSettlement: Date;                 // 마지막 일일 정산 시점
  };

  // 아레나 설정 (티어별 차등 크기)
  arena: {
    maxPlayers: number;     // 30
    mapWidth: number;       // 티어별: S=6000, A=5000, B=4000, C=3000, D=2500
    mapHeight: number;      // mapWidth와 동일 (정사각형)
  };
}

interface CountryRegions {
  countryCode: string;           // "KOR"
  tier: 'S' | 'A' | 'B' | 'C' | 'D';
  regions: Region[];             // 3~7개
  sovereignty: {
    controllingFactionId: string | null;  // 모든 지역 통일 시
    controlSince: Date | null;
  };
}
```

**티어별 아레나 크기** (v33의 8000px 기본 크기 대비 조정):

| 국가 티어 | 아레나 크기 (px) | 밀도 사유 | maxPlayers |
|----------|----------------|----------|------------|
| **S** | 6000 × 6000 | 강대국 = 넓은 전장, 전략적 깊이 | 30 |
| **A** | 5000 × 5000 | 지역 강국 = 중형 전장 | 30 |
| **B** | 4000 × 4000 | 중견국 = 적당한 밀도 | 25 |
| **C** | 3000 × 3000 | 소국 = 좁고 빠른 전투 | 20 |
| **D** | 2500 × 2500 | 도시국가 = 초밀집 즉발 교전 | 15 |

> **설계 근거**: v33 Arena 기준 8000px에서 30명이 플레이하면 밀도가 낮다. 티어별 축소하여 교전 빈도를 높이고, D티어 도시국가는 좁은 맵에서 즉각적인 팩션 충돌이 발생하도록 한다.

## 3. 게임 사이클 — 15분 라운드 (10분 PvE + 5분 배틀로얄)

### 3.1 PvE 파밍 페이즈 (10분)

기존 에폭 사이클(10분15초, 6페이즈)을 **15분 라운드 (2페이즈)**로 재설계한다.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                     15분 라운드 사이클 (1 Round = 900초)                      │
│                                                                              │
│ ┌──────────────────────────────────┐  ┌──────────────────────────────────┐  │
│ │ PvE FARMING PHASE (10분 = 600초)  │  │ BATTLE ROYALE PHASE (5분 = 300초) │  │
│ │                                    │  │                                    │  │
│ │ • 몬스터 사냥 (XP, Gold)           │  │ • 세이프존 수축 시작               │  │
│ │ • 고유 자원 채취                   │  │ • 팩션 vs 팩션 PvP 활성화         │  │
│ │ • PvP 비활성 (공격 불가)           │  │ • 같은 팩션 = 공격 면역            │  │
│ │ • 팩션원끼리 협동 파밍             │  │ • 최후 1팩션 생존 → 승리           │  │
│ │ • 빌드 강화 (레벨업 + 스킬)       │  │ • 존 밖 = 5% HP/s 감소            │  │
│ │ • 맵 전체 탐색 가능               │  │ • 에어드롭 (무기/버프 아이템)      │  │
│ │                                    │  │ • 팩션 아군 부활 불가              │  │
│ │ [9:00] "전쟁 1분 전!" 경고        │  │                                    │  │
│ │ [9:50] 10초 카운트다운             │  │ [13:00] 존 축소 2단계              │  │
│ │                                    │  │ [14:30] 존 축소 최종 (500px 반경)  │  │
│ └──────────────────────────────────┘  └──────────────────────────────────┘  │
│                                                                              │
│ ┌──────────────────────────────────────────────────────────────────────────┐ │
│ │ SETTLEMENT (15초) → 다음 라운드 시작                                     │ │
│ │ • 생존 팩션 상금 지급  • 지역 포인트 갱신  • 리더보드 업데이트            │ │
│ └──────────────────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────────────┘
```

**PvE 파밍 페이즈 상세 (0:00 ~ 10:00)**:

| 시간 | 이벤트 | 효과 |
|------|--------|------|
| 0:00 | 라운드 시작 | 모든 플레이어 맵 랜덤 위치에 스폰. 팩션별로 근처 스폰 |
| 0:00~3:00 | 초기 파밍 | Tier 1 몹, 기본 자원 노드 활성 |
| 3:00~6:00 | 중반 파밍 | Tier 2 몹 추가, 희귀 자원 노드 출현 |
| 6:00~9:00 | 후반 파밍 | Tier 3 엘리트 몹, 보스 몹 출현 (팩션 협동 필요) |
| 9:00 | 전쟁 경고 | "⚔️ 배틀로얄 1분 후 시작!" HUD 경고 |
| 9:30 | 팩션 집결 | 팩션원 위치 미니맵에 표시 (집결 유도) |
| 9:50 | 카운트다운 | 10→0 카운트다운, 긴장감 연출 |
| 10:00 | PvE 종료 | 남은 몹 전부 소멸, PvP 활성화 |

**자원 채취 메커니즘**:
- 맵 곳곳에 **자원 노드(Resource Node)** 배치 (30~50개/맵)
- 노드에 접근하면 **3초 채취 프로그레스바** → 자원 획득
- 채취 중 피격 시 캔슬 (PvE 중 몹 공격에 의해)
- 희귀 노드는 6분 이후 출현, 채취량 ×3
- 채취한 자원은 **개인 인벤토리** → 라운드 종료 시 **팩션 국고에 자동 입금**

### 3.2 배틀로얄 페이즈 (5분)

```
세이프존 수축 타임라인 (아레나 크기에 비례):
  10:00 (배틀 시작)  → 전체 맵 (티어별: S=6000, A=5000, B=4000, C=3000, D=2500)
  11:00 (1분 경과)   → 맵 대각선의 50% 반경 (1단계 수축)
  13:00 (3분 경과)   → 맵 대각선의 25% 반경 (2단계 수축)
  14:30 (4분30초)    → 반경 500px  (최종 수축, 티어 무관 동일)
  15:00 (라운드 종료) → 시간 종료 판정
```

**배틀로얄 핵심 규칙**:

| 규칙 | 설명 |
|------|------|
| **팩션 면역** | 같은 팩션 소속 플레이어 간 데미지 0 (투사체 관통) |
| **팩션 식별** | 팩션 컬러 링 + 팩션 태그 네임플레이트 |
| **세이프존 밖 DPS** | 5% maxHP / 초 (존 밖에서 30초 = 사망) |
| **부활 불가** | 배틀로얄 중 사망 = 관전 모드 전환 |
| **에어드롭** | 2분마다 맵 중심 근처에 파워업 드롭 (무기 강화, 일시 무적, 스피드 부스트) |
| **승리 조건** | 마지막까지 생존한 팩션 멤버가 1명 이상 있는 팩션이 승리 |
| **동시 전멸** | 존 수축으로 모든 팩션 전멸 시 → 마지막 사망 팩션 승리 |
| **시간 초과** | 15분 경과 시점에 2+ 팩션 생존 → 남은 팩션 중 생존 인원 수 + 총 킬 수 판정 |

**팩션별 시각 구분**:
- 각 팩션은 고유 **팩션 컬러** 보유 (최대 12색 팔레트)
- 플레이어 발 밑에 팩션 컬러 원형 링 표시
- 네임플레이트에 `[팩션태그] 닉네임` 형식
- 아군: 초록 네임 + 미니맵 초록 점 / 적군: 빨강 네임 + 미니맵 빨강 점

### 3.2.1 전략 거점 (Capture Points → Strategic Control Points)

v33의 Capture Point 시스템을 v39 라운드 사이클에 맞게 **전략 거점(Strategic Control Point)**으로 재설계한다.

**PvE 페이즈에서의 역할** (0:00 ~ 10:00):
- 맵에 3~5개 **전략 거점** 배치 (맵 크기에 비례)
- 거점 점령: 거점 내 5초 체류 → 점령 완료 (팩션 최초 1인 기준)
- 점령 효과: 해당 거점 반경 300px 내 **자원 채취 속도 ×2**, **XP 획득 +20%** 버프
- 경쟁: 여러 팩션이 동시 점령 시도 → 인원수 우세 팩션이 점령 (PvE 중에는 데미지 없이 인원 경쟁만)
- PvE 중 전투 없이 순수 위치 경쟁이므로, 팩션 간 **전략적 맵 통제** 요소 제공

**배틀로얄 페이즈에서의 역할** (10:00 ~ 15:00):
- PvE 때 점령한 거점이 **배틀로얄 보급 거점**으로 전환
- 거점 보유 팩션: 해당 위치에서 에어드롭 우선 배치 + 방어 보너스 (+5% 방어력)
- 세이프존 수축 시 **거점 위치가 세이프존 중심에 가까울수록 유리** (전략적 점령 보상)
- 거점 재탈취 가능: 적 팩션이 거점 내에서 해당 팩션 전원 처치 시 탈취

**v33 Capture Point와의 호환 매핑**:
| v33 Capture Point | v39 Strategic Control Point |
|-------------------|---------------------------|
| 에폭 중 점령 → Nation Score | PvE 중 점령 → 자원/XP 버프 |
| 점령 시 영역 효과 | 점령 시 반경 300px 버프 존 |
| 3개/맵 고정 | 3~5개/맵 (티어별 조정: S=5, A=4, B/C/D=3) |
| 단일 플레이어 점령 | 팩션 단위 점령 (인원수 기준) |

## 4. 팩션 전투 규칙 (Faction-Based PvP)

### 4.1 팩션 식별 & 동맹

기존 v35 팩션 시스템(3~50명, 4계층, 국고 6자원)을 **인게임 전투와 직결**시킨다.

```
팩션 전투 관계도:

  팩션 A ──── 아군 ────── 팩션 A 멤버
    │                        │
    │ (적대)                  │ (공격 면역)
    │                        │
  팩션 B ──── 아군 ────── 팩션 B 멤버
    │
    │ (적대 or 동맹)
    │
  팩션 C ──── 아군 ────── 팩션 C 멤버
```

**핵심 규칙**:
1. **같은 팩션 = 절대 공격 불가** (투사체 관통, 영역 효과 면역)
2. **동맹 팩션 = 공격 불가** (Military Alliance 조약 체결 시)
3. **나머지 = 적대** (배틀로얄 PvP 대상)
4. **무소속 = 1인 팩션** (팩션 미가입 시 자동으로 solo-faction 생성, 동맹 없음)

**팩션 식별 시스템**:

| 식별 요소 | 위치 | 설명 |
|----------|------|------|
| **팩션 컬러 링** | 캐릭터 발 아래 | 3px 원형 링, 팩션 컬러 |
| **팩션 태그** | 네임플레이트 위 | `[AWW] PlayerName` 형식 |
| **미니맵 색상** | 미니맵 | 아군 초록, 적군 빨강, 동맹 파랑 |
| **HP바 색상** | 캐릭터 위 | 아군 초록바, 적군 빨강바 |
| **팩션 배너** | HUD 좌상단 | 소속 팩션 엠블럼 + 생존 멤버 수 |

### 4.2 팩션 PvP 메커니즘

**데미지 매트릭스**:

| 공격자 → 피해자 | 같은 팩션 | 동맹 팩션 | 적대 팩션 | 몬스터 |
|----------------|----------|----------|----------|--------|
| **직접 공격** | 0% | 0% | 100% | 100% |
| **투사체** | 관통 (0%) | 관통 (0%) | 100% | 100% |
| **영역 효과** | 0% | 0% | 80% | 100% |
| **넉백** | 0% | 0% | 100% | 100% |

**PvP 킬 포인트**:

| 행동 | 개인 Gold | Nation Score | Region Point 기여 |
|------|----------|-------------|-------------------|
| PvP 킬 (적 팩션 처치) | +20 Gold | +15 Score | 생존 판정에 기여 |
| PvP 어시스트 (3초 내 데미지) | +8 Gold | +5 Score | - |
| 팩션 전멸 기여 (마지막 킬) | +50 Gold | +30 Score | - |
| 에어드롭 확보 | +15 Gold | - | - |

### 4.3 팩션 간 전술

배틀로얄에서 **팩션 팀전**이 되면 새로운 전술 차원이 열린다:

**1. 핀서 공격 (Pincer Attack)**
- 팩션원을 2개 조로 나눠 적을 양면에서 포위
- 한 조가 어그로 → 다른 조가 후방 급습

**2. 수비 진형 (Defensive Formation)**
- 탱커 캐릭터(Dozer, Tank)가 전면, 딜러(Neo, Trinity)가 후방
- 세이프존 수축 시 유리한 위치를 먼저 점거

**3. 킬 집중 (Focus Fire)**
- 팩션 리더가 타겟 지정 → 전원이 한 적 팩션에 집중
- 수적 우세 확보 후 다음 팩션 처리

**4. 자원 압박 (Resource Denial)**
- PvE 중 상대 팩션 근처 자원 노드를 먼저 채취
- 상대의 빌드 성장을 억제 (레벨/장비 차이로 배틀로얄 유리)

**5. 분산 생존 (Scatter Survival)**
- 팩션원을 맵 곳곳에 분산 배치
- 1명이라도 생존하면 팩션 승리 → 생존 확률 극대화

> **AI 에이전트 팩션 전술**: aww-agent-skill SDK의 NationAgent가 30초 주기로 팩션 전술을 결정할 수 있다. LLM이 "핀서 vs 수비 vs 분산" 중 최적 전략을 선택하는 것이 가능.

### 4.4 PvP 유형 구분: 배틀로얄 vs 전쟁 선포 (War Declaration)

v35의 **War State Machine** (Declaration → Prep 24h → Active 72h → End → Cooldown 24h)과 v39의 **라운드 배틀로얄**은 **독립적으로 공존하는 두 가지 PvP 레이어**다.

```
PvP 계층 구조:

  ┌─────────────────────────────────────────────────────────────┐
  │ Layer 1: 라운드 배틀로얄 (v39 — 지역 단위)                   │
  │ • 15분 라운드 내 자동 PvP (매 라운드 반복)                   │
  │ • 같은 지역 아레나 내 팩션 간 교전                           │
  │ • 보상: Region Points (+3/+1/+0) + Gold/Token                │
  │ • 규모: 소규모 (1개 아레나, 최대 30명)                       │
  │ • 조건: 없음 (라운드마다 자동 발생)                          │
  └─────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────┐
  │ Layer 2: 전쟁 선포 (v35 War System — 국가 단위)              │
  │ • 팩션 리더가 적대 팩션에 전쟁 선포 (외교 행동)              │
  │ • Prep(24h) → Active(72h max) → End → Cooldown(24h)         │
  │ • 보상: 패배 팩션 지역 즉시 탈취 + War Spoils (국고 약탈)   │
  │ • 규모: 대규모 (복수 지역 동시 교전, Cross-Arena Invasion)   │
  │ • 조건: 팩션 리더 선포 + 국고 자원 소모 (전쟁 비용)         │
  └─────────────────────────────────────────────────────────────┘
```

**전쟁 선포 시 배틀로얄과의 상호작용**:

| 상태 | 배틀로얄 | 전쟁 선포 효과 |
|------|---------|--------------|
| **평시** (War 없음) | 정상 진행, 모든 팩션 참가 | - |
| **전쟁 Prep** (24h) | 정상 진행 | 선포 팩션 아레나에 "⚔️ WAR INCOMING" 경고 HUD |
| **전쟁 Active** (72h) | 배틀로얄 PvP 데미지 **+20% 보너스** (전쟁 팩션 간) | Cross-Arena Invasion 가능: 전쟁 상대 팩션의 지역 아레나에 직접 침공 |
| **전쟁 End** | 정상 복귀 | 승리 팩션이 패배 팩션의 지배 지역 1개 즉시 탈취 |
| **전쟁 Cooldown** (24h) | 정상 진행 | 전쟁 재선포 불가, 방어 집중 기간 |

**Cross-Arena Invasion** (전쟁 Active 중 특수 메커니즘):
- 전쟁 Active 기간 중 공격 팩션 리더가 **"침공"** 버튼으로 적 팩션 지배 지역에 팩션원 전송
- 침공 팩션은 해당 지역의 **배틀로얄 페이즈에 참가** (외부 팩션으로 등장)
- 침공 팩션이 해당 지역에서 배틀로얄 1위 → 해당 라운드 Region Points를 **방어 팩션 대신 획득**
- 3일간의 전쟁 기간 중 누적 Region Points 비교로 전쟁 승패 결정

> **설계 근거**: 배틀로얄은 "일상적 영토 경쟁" (매일 96라운드), 전쟁 선포는 "전략적 대규모 공세" (최대 72시간). 두 레이어가 겹치면 전투 강도가 증폭되어 전쟁 기간이 게임 내 가장 긴장감 높은 이벤트가 된다.

## 5. 영토 지배 시스템

### 5.1 라운드 생존 → 지역 포인트

**생존 → 지역 포인트 → 일일 지배 → 국가 주권** 의 4단계 파이프라인.

```
라운드 생존 ──→ Region Points ──→ 일일 정산 ──→ 지역 지배 ──→ 국가 주권
  (15분)       (+3/+1/+0)       (24시간 집계)   (최다 포인트)  (모든 지역 통일)
```

**라운드 생존 순위 → Region Points**:

| 순위 | 조건 | Region Points | 보너스 |
|------|------|--------------|--------|
| **1위** | 마지막까지 생존한 팩션 | +3 RP | Gold ×1.5, Token ×2.0 |
| **2위** | 두 번째로 전멸한 팩션 | +1 RP | Gold ×1.2 |
| **3위 이하** | 그 이전에 전멸 | +0 RP | - |
| **참가 보너스** | 배틀로얄 참가한 모든 팩션 | +0.5 RP | 참가만으로 최소 포인트 |

> **하루 96라운드** (15분 × 96 = 24시간). 1위 독점 시 최대 288 RP/일.
> 현실적으로 3~5개 팩션이 경쟁하면 1위 팩션은 하루 ~60~100 RP 수준.

### 5.2 일일 지배 판정

**매일 UTC 00:00에 일일 정산**:

```
일일 정산 알고리즘:
  1. 각 지역별 dailyScores 집계
  2. 최고 점수 팩션 확인
  3. 2위와의 격차가 전체의 10% 이상이면 → 지배 확정
  4. 10% 미만 격차면 → "경합 중" (어느 팩션도 지배 안 됨)
  5. 지배 확정 시 → controllingFactionId 갱신
  6. dailyScores 리셋

예시 (한국 서울 지역):
  팩션 A: 85 RP (41.5%)
  팩션 B: 72 RP (35.1%)
  팩션 C: 48 RP (23.4%)

  격차: 41.5% - 35.1% = 6.4% < 10% → 경합 중!

  다음 날:
  팩션 A: 92 RP (48.4%)
  팩션 B: 61 RP (32.1%)
  팩션 C: 37 RP (19.5%)

  격차: 48.4% - 32.1% = 16.3% > 10% → 팩션 A 지배 확정!
```

**지배 유지**:
- 지배 확정 후에도 **매일 정산으로 재판정**
- 다른 팩션이 다음 날 더 많은 RP를 모으면 **지배 교체**
- **연속 지배 일수**에 따라 Sovereignty Escalation Ladder 단계 상승

**Sovereignty Escalation Ladder** (v35 호환):

v35의 3단계 주권 사다리를 v39의 일일 정산 모델에 통합한다. v35의 `None → Active Domination (6 epochs/1hr) → Sovereignty (24hr) → Hegemony (7 days)` 구조에서, v39에서는 에폭 단위가 아닌 **일 단위**로 재정의한다.

```
Sovereignty Escalation Ladder (지역 단위):

  ┌──────────────────────────────────────────────────────────────┐
  │ Tier 0: None (미지배)                                         │
  │ → 어느 팩션도 지배 미확정 또는 경합 중                        │
  │ → 효과: 없음                                                  │
  ├──────────────────────────────────────────────────────────────┤
  │ Tier 1: Active Domination (1일 연속 지배)                     │
  │ → 일일 정산에서 지배 확정 1회 이상                            │
  │ → 효과: 기본 지배 보상 (§5.4) + 팩션 깃발 표시               │
  │ → v35 매핑: Active Domination (6 epochs ≈ 1 day in v39)      │
  ├──────────────────────────────────────────────────────────────┤
  │ Tier 2: Sovereignty (3일 연속 지배)                           │
  │ → 동일 팩션이 3일 연속 지배 유지                              │
  │ → 효과: +수비대 NPC 1기, 방어 보너스 +5%                     │
  │ → v35 매핑: Sovereignty (24hr continuous → 3 days in v39)     │
  ├──────────────────────────────────────────────────────────────┤
  │ Tier 3: Hegemony (14일 연속 지배)                             │
  │ → 동일 팩션이 14일 연속 지배 유지                             │
  │ → 효과: "항구적 주권" — 교체 시 48시간 쿨다운 부여            │
  │ →        지역 요새화 (-10% 공격 디버프), 보상 ×1.5           │
  │ → v35 매핑: Hegemony (7 days → 14 days in v39)               │
  └──────────────────────────────────────────────────────────────┘

  * v35의 binary 모델(controllingFactionId만)에서 v39는
    sovereigntyTier: 0|1|2|3 필드를 추가하여 단계를 관리한다.
  * 지배 교체 시 → sovereigntyTier 즉시 0으로 리셋 (이전 지배 기간 무효)
```

```typescript
// v39 확장 — Region 인터페이스에 추가
interface RegionDominance {
  controllingFactionId: string | null;
  controlSince: Date | null;
  consecutiveDays: number;          // 연속 지배 일수 (0~30)
  sovereigntyTier: 0 | 1 | 2 | 3;  // None | Active | Sovereignty | Hegemony
  dailyScores: Record<string, number>;
  lastSettlement: Date;
}
```

### 5.3 국가 지배 (전체 지역 통일)

**한 팩션이 특정 국가의 모든 지역을 동시에 지배**하면 → **국가 주권(National Sovereignty)** 획득. 국가 주권은 개별 지역의 Sovereignty Tier와 별도로, **모든 지역을 통일**한 상위 레이어다.

```
예시: 한국 (5 지역)

  서울 수도권    → 팩션 A 지배 ✅
  경기 산업벨트  → 팩션 A 지배 ✅
  부산 항구      → 팩션 B 지배 ❌
  제주 자원지대  → 팩션 A 지배 ✅
  DMZ 전선      → 팩션 A 지배 ✅

  → 팩션 A는 4/5 지역 지배, 부산만 남음
  → 부산 탈환 시 국가 주권 획득!

  모든 지역 팩션 A:
  → 🏆 팩션 A가 대한민국 주권(Sovereignty) 획득!
```

**국가 주권 효과** (기존 v33/v35 시스템과 통합):

| 효과 | 수치 | 설명 |
|------|------|------|
| XP 부스트 | +15% | 해당 국가 아레나에서 XP 획득 증가 |
| Speed 부스트 | +5% | 이동 속도 증가 |
| 캡처 속도 | +20% | 자원 노드 채취 속도 증가 |
| 토큰 보상 배율 | ×1.3 | 해당 국가 토큰 보상 30% 증가 |
| 거버넌스 권한 | 정책 설정 | 해당 국가의 정책(세율, 교역 규칙 등) 결정 |

### 6.2 채취 메커니즘

**맵 내 자원 노드 시스템**:

```

│ 맵 (8000 × 8000)                                │
│                                                  │
│   ◆ 자원 노드 (30~50개/맵)                      │
│   ◆ = 일반 노드 (항상 존재, 3초 채취, 1~3 자원)  │
│   ★ = 희귀 노드 (6분 후 출현, 5초 채취, 5~10 자원)│
│   ⬟ = 특산 노드 (8분 후 출현, 8초 채취, 1 특산)  │
│                                                  │
│   ◆     ◆        ◆                              │
│       ◆     ◆         ◆                         │
│   ◆        ★      ◆                             │
│       ◆        ◆     ◆    ⬟                     │
│   ◆     ◆     ◆        ◆                        │
│              ◆    ◆         ◆                    │
└────────────────────────────────────────────────┘
```

**채취 프로세스**:
1. 플레이어가 자원 노드 반경 50px 이내 접근

3. 채취 중 피격 시 → 캔슬 (PvE 몹 or 배틀로얄 시 적 PvP)
4. 완료 시 → 자원 획득 + 노드 고갈 (60초 후 재생)
5. 팩션원이 동시 채취 시 → 속도 +30%/명 (협동 보너스)

**자원 캐리 규칙**:
- 채취한 자원은 **개인 인벤토리**에 보관 (최대 50 단위)
- PvE 페이즈 중 사망 시 → 50% 드롭 (다른 팩션이 줍기 가능)
- 배틀로얄 중 사망 시 → 100% 드롭 (적 팩션에게 뺏김!)
- 라운드 생존 시 → 전량 팩션 국고에 자동 입금

## 7. 확장된 게임 디자인 (사용자가 고려하지 못한 영역)

### 7.1 시즌 시스템 & 리셋 정책

기존 4주 시즌(Discovery→Expansion→Empires→Reckoning) 시스템과 영토 지배의 연동:

| 시즌 Era | 기간 | 영토 시스템 효과 |
|----------|------|-----------------|
| **Discovery** (1주차) | 1~7일 | 지역 지배 비용 ×0.5 (빠른 확장 유도) |
| **Expansion** (2주차) | 8~14일 | 지역 수 ×1.5 (숨겨진 지역 해금) |
| **Empires** (3주차) | 15~21일 | 국가 주권 보상 ×2 (대규모 충돌) |
| **Reckoning** (4주차) | 22~28일 | 모든 지배 연속 보너스 리셋, 최종 쟁탈전 |

**시즌 종료 리셋 정책**:
- ✅ 리셋: 모든 지역/국가 지배 상태, 연속 지배 보너스, NPC 수비대
- ✅ 리셋: 레벨, 빌드, 팩션 프레스티지 순위
- ❌ 보존: 팩션 존재, 멤버십, 국고 자원, 온체인 토큰
- ❌ 보존: 기술 트리 해금 (시즌 간 유지)
- 🏆 시즌 MVP: 가장 많은 국가 주권을 보유한 팩션 → **시즌 챔피언 칭호** + $AWW 대량 보상

### 7.2 팩션 밸런싱 — 수적 불균형 방지

**핵심 문제**: 50명 팩션 vs 5명 팩션이 같은 지역에서 배틀로얄을 하면 공정하지 않다.

**해결책 — 스케일링 메커니즘**:

| 메커니즘 | 설명 |
|---------|------|
| **역수 보정 (Underdog Boost)** | 배틀로얄 시작 시 인원수 비례 역보정. 5명 팩션은 개인당 HP +30%, 데미지 +20% |
| **지역 아레나 인원 상한** | 지역당 30명 상한. 한 팩션이 20명 참가하면 나머지 팩션은 합산 10명까지 |
| **팩션 인원 상한/지역** | 한 팩션이 한 지역에 최대 10명까지만 참가 가능. 나머지는 다른 지역 배치 |
| **NPC 지원군** | 3명 이하 팩션에 NPC 아군 2명 자동 배치 (기본 전투력, 플레이어보다 약함) |
| **Region Point 가중치** | 소규모 팩션(10명 이하)의 RP ×1.5 (지배 경쟁 촉진) |

**제르그(Zerg) 방지**:
- 한 팩션이 50명 전원을 같은 지역에 보내는 것 방지
- **팩션 인원/지역 상한 = max(10, total_members / regions_count)**
- 큰 팩션은 여러 지역에 분산 배치 강제 → 전략적 자원 배분 필요

### 7.3 오프라인 방어 — 24시간 지배 유지 문제

**핵심 문제**: 팩션 멤버가 잠든 시간대(8시간)에 적 팩션이 라운드를 독식하면 지배를 잃는다.

**해결책 — 다중 방어 메커니즘**:

| 메커니즘 | 설명 |
|---------|------|
| **NPC 수비대** | 지배 팩션이 Gold+Food로 NPC 수비대를 배치. 배틀로얄에 자동 참전 (플레이어의 60% 전투력) |
| **시간대 가중치** | 팩션의 "활동 시간대" 설정 (예: UTC+9 18:00~24:00). 활동 시간대의 라운드 RP ×2 |
| **지배 관성** | 연속 3일+ 지배 시, 지배 교체에 필요한 격차가 10% → 20%로 증가 |
| **동맹 방어** | Military Alliance 체결한 동맹 팩션의 멤버가 배틀로얄에서 지배 방어에 참여 가능 |
| **AI 에이전트 배치** | aww-agent-skill SDK로 배치한 AI 에이전트가 24시간 자동 전투 (핵심 방어 수단!) |

> **AI 에이전트가 핵심**: 24시간 방어의 가장 효과적인 수단은 **AI 에이전트를 배치**하는 것. 팩션은 에이전트 빌드를 훈련시키고 배치하여 오프라인 시간을 커버한다. 이것이 **aww-agent-skill SDK의 존재 이유**이자 **P2E의 핵심 동력**.

### 7.4 용병 & 고용 시스템

팩션 인원이 부족할 때 **용병 NPC**를 고용하여 전투력을 보충한다.

| **전설 용병** | Gold 1000 + Oil 100 | 플레이어 100% | 1일 | 스킬 3개 + 지능형 AI |

### 7.6 연맹(Coalition) & 대규모 전쟁

**연맹**: 여러 팩션이 모여 **초국가적 동맹**을 형성하는 시스템.

```
연맹 구조:
  ┌──────────────────────────────────────┐
  │ COALITION (연맹)                       │
  │ 최대 5개 팩션, 최소 3개 팩션           │
  │                                        │
  │ ┌──────┐ ┌──────┐ ┌──────┐            │
  │ │팩션 A│←→│팩션 B│←→│팩션 C│            │
  │ │(리더)│  │(멤버)│  │(멤버)│            │
  │ └──────┘ └──────┘ └──────┘            │
  │                                        │
  │ 공유: 영토 방어, 자원 교역, 전쟁 참전   │
  │ 제한: 같은 배틀로얄에서는 각각 독립 팩션│
  └──────────────────────────────────────┘
```

**연맹 규칙**:
- 연맹 내 팩션은 **배틀로얄에서는 독립** (같은 연맹이라도 서로 공격 가능!)
- 다만 **영토 지배 방어**에서는 협력 (연맹 팩션의 지배 영토도 공동 방어)
- **대규모 전쟁**: 연맹 vs 연맹 전쟁 선포 가능 → 모든 소속 팩션 간 적대

## 9. 기존 시스템과의 통합

### 9.1 v33 에폭 시스템 확장

기존 6페이즈 에폭 사이클을 **3페이즈 라운드 사이클**로 대체한다.

| 기존 에폭 페이즈 | 시간 | 신규 매핑 |
|----------------|------|----------|
| Peace (4:50) | 290초 | → **PvE Phase** (600초로 확장) |
| War CD (0:10) | 10초 | → PvE 마지막 10초 카운트다운으로 통합 |
| War (3:00) | 180초 | → **Battle Royale Phase** (300초로 확장) |
| Shrink (2:00) | 120초 | → BR 내 세이프존 수축으로 통합 |
| End (0:05) | 5초 | → **Settlement Phase** (15초로 확장) |
| Transition (0:10) | 10초 | → Settlement에 포함 |

**마이그레이션 전략**: `EpochManager`를 **deprecated** 처리하고 새로운 `RoundEngine`을 생성한다. 기존 v33 모드는 레거시 호환을 위해 유지하되, v39 지역 아레나는 반드시 `RoundEngine`을 사용한다.

```
마이그레이션 경로:

  Phase 1 (v39 초기):
    EpochManager (v33, deprecated) — 기존 Matrix/Singularity 모드에서 계속 사용
    RoundEngine  (v39, 신규)       — Region Arena 전용으로 병행 운영

  Phase 2 (v39 안정화 후):
    EpochManager → RoundEngine 어댑터 추가 (EpochManager 호출 → 내부 RoundEngine 위임)
    기존 모든 모드가 RoundEngine 기반으로 전환

  Phase 3 (v40 이후):
    EpochManager 완전 제거
    RoundEngine 이 유일한 게임 사이클 엔진
```

**DominationEngine 변환 (v35 6-epoch → v39 4-round)**:

v35의 `Active Domination` 진입 조건이 "6 epochs (약 1시간)"인데, v39에서는 epoch 대신 round를 사용한다. 1 round = 15분이므로:

| v35 기준 | v35 에폭 수 | v39 라운드 수 | v39 실시간 |
|---------|------------|-------------|----------|
| Active Domination 진입 | 6 epochs | **4 rounds** | ~1시간 (4 × 15분) |
| 일일 정산 주기 | ~60 epochs | **96 rounds** | 24시간 |

```go
// 기존 (v33) — DEPRECATED, v39 이후 사용 금지
const (
    EpochPhasePeace       = iota  // 290s
    EpochPhaseWarCountdown        // 10s
    EpochPhaseWar                 // 180s
    EpochPhaseShrink              // 120s
    EpochPhaseEnd                 // 5s
    EpochPhaseTransition          // 10s
)

// 신규 (v39) — RoundEngine
const (
    RoundPhasePvE         = iota  // 600s (10분)
    RoundPhaseBattleRoyale        // 300s (5분)
    RoundPhaseSettlement          // 15s
)

// RoundEngine — EpochManager를 대체하는 신규 사이클 엔진
type RoundEngine struct {
    phase        int
    phaseStart   time.Time
    roundNumber  int
    regionID     string
    onPhaseChange func(phase int, timeLeft int)
}

// EpochAdapter — 기존 EpochManager 호출을 RoundEngine으로 위임 (Phase 2 마이그레이션)
type EpochAdapter struct {
    engine *RoundEngine
}
func (a *EpochAdapter) GetCurrentPhase() int {
    // EpochPhase 상수를 RoundPhase로 변환
    switch a.engine.phase {
    case RoundPhasePvE:          return EpochPhasePeace
    case RoundPhaseBattleRoyale: return EpochPhaseWar
    case RoundPhaseSettlement:   return EpochPhaseEnd
    default:                     return EpochPhasePeace
    }
}
```

> **핵심 결정**: v33의 compile-time tick 상수(20Hz)는 RoundEngine에서도 유지한다. 변경되는 것은 페이즈 정의와 타이밍뿐이며, 게임 루프 틱 레이트는 그대로 20Hz다.

### 9.2 v35 토큰 이코노미 연결

**보상 파이프라인 변경**:

```

신규 (v39):
```
