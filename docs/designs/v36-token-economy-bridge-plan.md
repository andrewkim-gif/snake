# PLAN: 인게임-아웃게임 토큰 이코노미 브릿지 — v36

> v36 — 인게임 재화 획득 → 아웃게임 세계 경제 활용의 완전한 순환 경제 설계
> 작성일: 2026-03-10

## 1. 개요

AI World War는 인게임에서 전투·생존·전략을 통해 재화를 벌고, 이를 아웃게임(로비)의 세계 경제에서 **실질적으로 사용**할 수 있는 완전한 순환 경제를 필요로 한다.

**현재 문제**: 토큰을 "받기만" 하는 구조. 서버에 TokenRewardManager, BuybackEngine, DefenseOracle이 완성되어 있으나 실제 호출이 없고 (dead code), 클라이언트는 보상 이벤트를 수신하지 않으며, 플레이어가 토큰을 **쓸 이유**가 명확하지 않다.

**이 기획의 목표**:
1. 인게임 전투 행위 → 재화 획득 파이프라인 완성 (Earn)
2. 인게임 재화 ↔ 온체인 토큰 교환 메커니즘 정립 (Bridge)
3. 아웃게임에서 토큰을 소비할 수 있는 다층적 활용처 설계 (Spend)
4. 인플레이션 제어 + 게임 밸런스 보호 안전장치 (Safety)
5. **사용자가 생각하지 못한 확장 영역**까지 선제적으로 설계

**핵심 키워드**: `Play → Earn → Bridge → Spend → Play` 무한 루프

**$AWW 토큰**: `0xfD486ba056dFa2d8625a8bB74e4f207F70ab8CD7` (CROSS Mainnet, chainId 8851)
**Forge Pool**: `https://x.crosstoken.io/forge/token/0xfD486ba...`

## 3. 핵심 설계 원칙

1. **Play First, Earn Second**: 토큰 획득은 재미있는 게임플레이의 부산물이어야 한다. 토큰 파밍이 최적 전략이 되면 안 된다.
2. **Soft Currency → Hard Currency 게이트**: 인게임 재화(Gold, RP)는 무한히 발행 가능한 soft currency. 온체인 토큰($AWW, 국가토큰)은 공급 한정 hard currency. 둘 사이에 반드시 **변환 게이트**가 존재해야 한다.
3. **Sink ≥ Faucet**: 토큰이 생성되는 속도보다 소멸/잠금되는 속도가 같거나 빨라야 한다. 디플레이셔너리 압력 우선.
4. **온체인은 최종 결산만**: 매 틱마다 온체인 트랜잭션을 날리지 않는다. 인게임은 서버 인메모리로 운영, 시즌/에포크 단위로 온체인 정산.
5. **경제적 행위 = 전략적 선택**: 모든 소비 행위는 게임플레이에 실질적 이점을 제공하되, "돈으로 이기는 게임(P2W)"이 되지 않도록 **캡과 카운터** 설계.
6. **보안 우선**: v30 검증에서 발견된 67개 이슈 (16 Critical) 수정 없이 토큰 순환 활성화 금지.
7. **점진적 탈중앙화**: Phase 1은 서버 중심 관리, Phase 2에서 온체인 정산 도입, Phase 3에서 컨트랙트 거버넌스 이양.

### 4.2 The Micro Loop (매치 내 순환)

하나의 에포크(10분) 안에서도 미니 경제가 순환한다:

```
오브 수집(3-10 XP) → 레벨업 → 더 강한 전투 → 킬 보상(Gold + NationScore)
                                                     │
                  인매치 Gold 소비 ◄──────────────────┘
                  (※ 현재 미구현 — 아래 5.3에서 설계)
```

**새로운 설계**: 인매치 Gold를 소비할 수 있는 **전장 상점(Field Shop)** 도입.
- 전장 상점: 에이전트가 특정 맵 오브젝트에 접근 시, Gold로 일시 버프/소모품 구매
- 리스폰 보험, 오브 자석, 임시 스텟 부스트 등
- 매치 종료 시 잔여 Gold의 일부(30%)가 Nation Score에 가산 → 토큰 보상으로 전환

## 5. 인게임 재화 체계 (Earn Layer)

### 5.1 재화 체계 정리 (3-Tier Currency Model)

| Tier | 재화 | 성격 | 발행 | 소멸 | 범위 |
|------|------|------|------|------|------|
| **T1 (Match)** | XP, Gold | 매치 내 소프트 통화 | 무한 (행위당 지급) | 매치 종료 시 리셋 | 단일 매치 |
| **T2 (Account)** | RP, Cosmetic Coin, Account XP | 계정 누적 소프트 통화 | 매치 성과 기반 | 코스메틱 구매로 소멸 | 영구 (서버 DB) |
| **T3 (Chain)** | $AWW, 국가 토큰 | 온체인 하드 통화 | Epoch/Season 정산 시 | 소각/잠금/거래 | 영구 (블록체인) |

**핵심**: T1 → T2 → T3로 **상향 변환**만 가능. 역방향(T3→T1) 불가.
이렇게 해야 온체인 토큰이 인매치 밸런스를 파괴하지 않는다.

### 5.2 T1→T2 변환: 매치 성과 → 계정 보상

매치 종료 시 변환 공식:

```
Account_XP = kills×5 + assists×2 + levels×3 + survivalMin×1 + objectives×2 + 10(참여) + MVP보너스
RP = 참여(5) + Top50%(10) + Top3(25) + Win(50) + Synergy(10) + HiddenSynergy(100) + kills≥3(5)
Cosmetic_Coin = levelUp보상 (10/레벨, 매5레벨 +25)
Gold_Carry = min(잔여Gold × 0.3, 500) → Nation Score 가산
```

### 5.3 전장 상점 (Field Shop) — 신규 설계

**목적**: 인매치 Gold에 **즉각적 소비 동기** 부여. 현재 Gold는 쌓이기만 하고 쓸 데가 없다.

| 아이템 | Gold 비용 | 효과 | 지속 |
|--------|----------|------|------|
| 오브 자석 (Orb Magnet) | 100 | 수집 반경 2배 | 30초 |
| 긴급 회복 (Quick Heal) | 150 | HP 50% 즉시 회복 | 즉시 |
| 스텟 주사 (Stat Injection) | 200 | 공격력 +25% | 20초 |
| 은폐 (Cloak) | 300 | 5초간 투명 (미니맵에서도) | 5초 |
| 리스폰 보험 (Respawn Ins.) | 500 | 사망 시 1회 즉시 리스폰 | 다음 사망 1회 |
| 국가 기여 (Nation Donate) | Any | Gold의 50%를 Nation Score로 변환 | 즉시 |

- **구매 위치**: 맵에 3개 Field Shop NPC 배치 (XP Shrine과 유사한 맵 오브젝트)
- **구매 제한**: 에포크당 최대 5회 구매 (봇 파밍 방지)
- **경제 효과**: Gold의 즉각적 소멸 → T1 레벨의 자체 순환 생성

### 5.4 T2→T3 변환: Nation Score → 토큰 보상 (기존 dead code 활성화)

기존 `TokenRewardManager`의 5가지 보상 채널을 **활성화**:

| 보상 유형 | 입력 | 출력 | 주기 | 기존 코드 |
|----------|------|------|------|----------|
| Domination | Nation Score | 국가 토큰 | 에포크(10분) | `QueueDominationReward` ✓ |
| Hegemony | 7일 지배 | $AWW | 주간 | `QueueHegemonyReward` ✓ |
| Sovereignty | 24시간 주권 | 국가 토큰 | 일간 | `QueueSovereigntyReward` ✓ |
| Epoch MVP | 에포크 1등 | 1.5배 배율 | 에포크 | `QueueEpochMVP` ✓ |
| War Victory | 전쟁 승리 | 2배 배율 | 이벤트 | `QueueWarVictoryReward` ✓ |

**활성화에 필요한 작업**:
1. `domination.go`의 에포크 종료 시 `QueueDominationReward()` 호출 연결
2. `matrix_engine.go`의 에포크 보상 시 `QueueEpochMVP()` 호출 연결
3. WebSocket `token_reward` 이벤트 클라이언트 수신 핸들러 구현
4. REST `/api/v14/rewards/{id}` 클라이언트 fetch 구현

### 5.5 RP 기반 추가 보상 — 신규 설계

RP(Reputation Point)는 현재 능력 슬롯 해금에만 쓰인다. 토큰 경제와 연결:

| RP 마일스톤 | 보상 |
|------------|------|
| 500 RP 누적 | $AWW 50 에어드롭 (1회성) |
| 1000 RP 누적 | 시즌 보상 배율 +10% 영구 |
| 2500 RP 누적 | 국가 토큰 데일리 보너스 해금 (+2 토큰/일) |
| 5000 RP 누적 | 거버넌스 투표권 2배 가중치 |
| 10000 RP 누적 | "전설" 칭호 + 시즌 보상 배율 +25% |

## 6. 브릿지 메커니즘 (Bridge Layer)

### 6.1 정산 방식 비교

| 방식 | 가스비 | 보안 | 실시간성 | 채택 |
|------|--------|------|---------|------|
| 매 틱 온체인 트랜잭션 | 극도로 높음 | 높음 | 실시간 | ❌ 비용 폭발 |
| 에포크 단위 배치 정산 | 중간 | 중간 | 10분 지연 | ✅ 기본 모드 |
| 시즌 단위 일괄 정산 | 매우 낮음 | 낮음 | 4주 지연 | ⚠️ 대형 보상만 |
| CROSS Ramp 크레딧 교환 | 0 (Ramp 수수료) | Ramp 의존 | 즉시 | ✅ 소액 교환 |

**결정**: **하이브리드 정산** — 소액은 인메모리 누적 후 일일 배치, 대형 보상은 에포크 단위 즉시 정산.

### 6.2 인게임 포인트 ↔ 온체인 토큰 교환 게이트

```
인게임 AWW 포인트 (서버 메모리)
       │
       ▼  [출금 요청]
  ┌─────────────────────┐
  │   Settlement Gate    │
  │                     │
  │  ① 최소 출금량: 100 AWW │
  │  ② 일일 출금 한도: 5000  │
  │  ③ 수수료: 5% (소각)    │
  │  ④ 쿨다운: 24시간       │
  │  ⑤ 계정 레벨 ≥ 5 필요  │
  │  ⑥ 봇 탐지 통과 필수    │
  └─────────┬───────────┘
            ▼
  온체인 $AWW 토큰 (지갑)
```

**역방향 (온체인 → 인게임)**:
- CROSS Ramp `burn` 주문: 온체인 $AWW → 인게임 포인트
- 즉시 반영 (Ramp webhook `order-result`)
- **용도**: 인게임 소비처(전쟁 선포, GDP 부스트 등) 이용

### 6.3 국가 토큰 브릿지

국가 토큰은 $AWW와 다른 정산 주기:

| 이벤트 | 정산 | 대상 토큰 |
|--------|------|----------|
| 에포크 도미네이션 | 에포크 종료 (10분) | 해당 국가 토큰 |
| 일일 주권 보상 | 매 24시간 | 주권 국가 토큰 |
| 전쟁 승리 보상 | 전쟁 종료 시 | 승리 국가 토큰 |
| 시즌 최종 보상 | 시즌 종료 | $AWW + 국가 토큰 |

**195개 국가 토큰 배포 전략** (Phase 3):
1. S-tier 7개국 우선 배포 (forge_token_deploy 사용)
2. A-tier 20개국 배포
3. 나머지는 시즌 진행에 따라 순차 배포 (해당 국가에 주권 달성 시 자동 배포)
4. **Lazy Deployment**: 아무도 플레이하지 않는 국가의 토큰은 배포하지 않음 → 가스비 절약

### 6.4 CROSS Ramp 크레딧 경로

기존 Ramp 웹훅이 지원하는 양방향 교환:

```
[플레이어] ──(인게임 크레딧)──→ [Ramp mint] ──→ [$AWW 온체인 토큰]
[플레이어] ◄──(인게임 크레딧)── [Ramp burn] ◄── [$AWW 온체인 토큰]
```

**수정 필요**: 현재 Ramp webhook에 HMAC 서명 검증이 v30에서 지적됨 → Phase 1에서 보안 수정 필수.

## 9. 사용자가 놓치기 쉬운 확장 영역

### 9.1 🔮 에이전트 마켓플레이스 (Agent Marketplace)

현재 에이전트 트레이닝은 개인 빌드 프로필 설정에 국한됩니다. 확장 가능한 경제:

- **빌드 레시피 NFT**: 승률 높은 빌드 조합을 NFT로 민팅 → 마켓에서 $AWW로 거래
- **에이전트 대리 전투**: 자신의 에이전트를 타인에게 대여 → 수익의 30% 수수료
- **에이전트 토너먼트**: 입장료 $AWW → 승자 독식 풀
- **왜 중요한가**: AI 에이전트 플랫폼의 핵심 차별점. 에이전트 자체가 거래 가능한 자산이 됨.

### 9.2 🌍 국가 간 무역 시스템 (Inter-Nation Trade)

v11 설계의 Export Settlement(20% exporter 토큰)을 확장:

- **자원 거래소**: 6가지 자원(석유, 광물, 농산물, 기술, 문화, 군사)을 국가 토큰으로 거래
- **무역 루트**: 동맹국 간 자동 무역 경로 → 양국 GDP 부스트
- **관세 시스템**: 비동맹국 무역에 10% 관세 → 국가 Treasury에 적립
- **경제 봉쇄**: 전쟁 시 적국 무역 차단 → 해당 국가 GDP 디버프
- **왜 중요한가**: 국가 토큰에 **실질적 수요**를 만든다. 자원이 필요하면 해당 국가 토큰을 사야 함.

### 9.3 ⚔️ 용병 시스템 (Mercenary System)

국가 경계를 넘는 플레이어 경제:

- **용병 계약**: 약소국이 $AWW로 강국 플레이어를 고용 (1 에포크 단위)
- **이중 보상**: 용병은 고용 국가 + 본국 양쪽에서 보상 (각 50%)
- **배신 메커니즘**: 전쟁 중 용병은 계약 파기 가능 (보증금 소각)
- **왜 중요한가**: 소국 플레이어도 경제적 가치를 가짐. 국가 간 인력 시장 형성.

### 9.4 📊 예측 시장 (Prediction Market)

게임 내 이벤트에 대한 베팅 시장:

- **전쟁 결과**: "USA vs CHN 전쟁에서 누가 이길까?" → $AWW 스테이킹
- **시즌 챔피언**: "이번 시즌 최종 1등 국가는?" → 시즌 시작 시 베팅
- **에이전트 성과**: "이 에이전트가 다음 에포크 MVP?" → 소액 베팅
- **오즈 산정**: AMM(Automated Market Maker) 기반 자동 배당률
- **왜 중요한가**: 관전자/비전투 플레이어에게도 경제 참여 기회. 게임 이벤트의 금융상품화.

### 9.5 🏛️ 국가 재정 시스템 (National Fiscal Policy)

거버넌스 투표로 결정되는 국가 경제 정책:

- **세율 조정**: 국가 내 거래 수수료 0-10% (높으면 Treasury 성장, 낮으면 플레이어 유입)
- **인프라 투자**: Treasury에서 $AWW 지출 → 국가 영구 버프 (GDP +5%, 방어 +5% 등)
- **보조금 정책**: 신규 플레이어에게 국가 토큰 에어드롭 (Treasury 지출)
- **국채 발행**: 국가가 $AWW를 차입 → Treasury로 이자 상환
- **왜 중요한가**: 거버넌스에 **실질적 의사결정**을 부여. 투표가 경제적 결과를 만듦.
