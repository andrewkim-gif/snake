## 1. 버전 간 모순 (Cross-Version Contradictions)

### VC-01 [CRITICAL] v14 EpochManager vs v15 RoomManager 아키텍처 충돌

- **v14**: "Room state machine → **replaced** by EpochManager" (v14-system-architecture.md §4.1)
- **v15**: "기존 5룸 + 국가별 아레나 관리 (에이전트 모드 확장)" — Room Manager 재사용 전제
- **충돌**: v14는 Room을 완전히 제거하고 EpochManager로 교체. v15는 Room이 존재한다고 가정.
- **영향**: v15 에이전트가 참가하는 "Room"이 실제로는 존재하지 않음. 전체 연결 계층 불일치.
- **수정안**: v15 문서에서 Room → CountryArena + EpochManager로 용어 통일. `agent_stream.go`가 EpochManager와 통신하도록 재설계.

### VC-02 [CRITICAL] v14 무기 시스템 vs v11 AgentStrategy 인터페이스 파괴

- **v11**: `AgentStrategy.build_priority.preferred_tomes: ("Blade" | "Aegis" | "Venom" | ...)[]`
- **v14**: Tome/Ability 시스템 → **replaced** by WeaponSystem + PassiveType + SynergyType
- **충돌**: v11의 Agent API 계약이 v14에서 완전히 깨짐. 외부 에이전트가 "Blade" Tome을 요청하면 서버가 이해 불가.
- **영향**: v15 외부 에이전트 연동 시 API 스키마 호환 불가. OpenClaw Skill이 잘못된 인터페이스 참조.
- **수정안**: AgentStrategy를 v14 무기 체계로 업데이트:
  ```typescript
  preferred_weapons: WeaponType[];  // "BonkMallet" | "ChainBolt" | ...
  preferred_passives: PassiveType[]; // "Vigor" | "Fury" | ...
  synergy_target: SynergyType | null;
  ```

### VC-03 [CRITICAL] v14 "NG-1: No blockchain changes" vs 실제 경제 영향

- **v14 비목표**: "NG-1: Blockchain token changes — Reuse v11 infrastructure (ADR-033)"
- **현실**: v14는 전투 시간 5분→10분, 리스폰 도입, 무기 시스템 교체 → Gold 생산량/분배 구조가 완전히 바뀜
- **충돌**: "5% GDP 세수 바이백"의 Gold가 v14 전투에서 얼마나 생성되는지 재계산 필요. v11 기준 5분 1회 전투 보상과 v14 10분 리스폰 데스매치 보상은 구조적으로 다름.
- **영향**: 토큰 바이백 속도가 설계와 다르게 작동 → 경제 밸런스 예측 불가
- **수정안**: v14 전투 보상 공식 확정 후 바이백 파라미터 재계산. Gold/분 생산율 시뮬레이션 필수.

### VC-04 [HIGH] v10 점수 공식 vs v14 리스폰 모순

- **v10/v11**: `생존 = Base 100점 + (킬 × 15) + (레벨 × 10)` — "1 Life" 전제, 생존 자체가 가치
- **v14**: 리스폰 데스매치 — 모든 에이전트가 "생존"함 (죽어도 부활)
- **충돌**: Base 100점 "생존 보너스"가 무의미해짐. 모든 참가자가 Base 100을 받으므로 차별화 요소 상실.
- **수정안**: v14 점수 공식을 K/D 비율 + 캡처 포인트 + 데미지 기여도 중심으로 재설계. "생존 보너스" 제거 또는 "에포크 내 사망 0회" 조건부로 변경.

### VC-05 [HIGH] Defense Oracle 5분 주기 vs v14 10분 에포크

- **v11**: Defense Oracle 5분마다 업데이트 (전투 사이클과 동기화)
- **v14**: 에포크 10분 (Peace 5분 + War 5분)
- **충돌**: Peace Phase 시작 시 계산된 방어 버프가 War Phase 도중 변경될 수 있음. 전투 시작 조건과 실제 적용 버프가 다를 수 있음.
- **수정안**: Defense Oracle을 에포크 시작 시 1회 스냅샷으로 변경. 에포크 중 버프 변경 없음 (전투 공정성 보장).

### VC-06 [MEDIUM] v14 "Capture Point" vs v11 "Faction Score" 이중 지배권 결정

- **v14**: 에포크 내 Capture Point 점령으로 국가 점수 결정 (§4.1 EpochManager.nationScores)
- **v11**: 팩션 점수 합산으로 지배권 결정 (§5.1 "팩션 점수 = Σ 개인 점수")
- **충돌**: 두 시스템의 "승리 조건"이 다름. Capture Point를 많이 점령한 팩션과 킬 수가 많은 팩션이 다를 수 있음.
- **수정안**: 단일 점수 공식으로 통합. 예: `NationScore = CapturePoints × 2 + Kills × 15 + LevelSum × 10`

## 2. 경제 모델 결함 (Economic Model Flaws)

### EF-01 [CRITICAL] 인게임 Gold → 온체인 바이백 브릿지 미정의

- **설계**: "Gold 거래의 5% → 재무부가 DEX에서 자국 토큰 매수" (v11 §15.3)
- **문제**: Gold는 **인게임 가상 자원**이고, DEX 바이백은 **온체인 실제 거래**. 두 경제를 연결하는 브릿지 메커니즘이 정의되지 않음.
  - 인게임 Gold 1개 = $AWW 몇 개?
  - 누가 실제 온체인 토큰을 매수하는가? (게임 서버? Treasury 봇?)
  - 매수 자금은 어디서 오는가? Treasury가 초기에 보유한 토큰을 매도해서 $AWW를 확보?
- **영향**: 토큰 이코노미의 핵심 메커니즘이 실행 불가능. "선순환 루프"의 첫 단계가 끊김.
- **수정안**: 명시적 브릿지 설계 필요:
  - **옵션 A**: 인게임 Gold → $AWW 교환 레이트 고정 (중앙화, 단순)
  - **옵션 B**: 유저가 Gold를 $AWW로 "인출" → 인출된 $AWW의 일부가 자동 바이백 (탈중앙화, 복잡)
  - **옵션 C**: 게임 재무부가 $AWW 기반 유동성을 보유하고, GDP 비례 자동 바이백 실행 (추천)

### EF-02 [CRITICAL] Death Spiral 플로어 부재

- **설계**: GDP↓ → 바이백↓ → 토큰 가격↓ → 방어↓ → 더 많이 정복당함 → GDP 더↓
- **문제**: 하향 나선(Death Spiral)을 차단하는 플로어 메커니즘이 없음. 한 번 약해진 국가는 계속 약해짐.
- **v11 기존 지적**: H-04에서 "최소 바이백 플로어" 제안했으나 구체적 수치/메커니즘 미정.
- **영향**: D등급 소국은 시즌 초반 한 번 점령당하면 회복 불가. 게임 다양성 상실.
- **수정안**:
  1. **최소 방어 버프 +5%** (토큰 가격과 무관한 기본 방어)
  2. **언더독 복구 메커니즘**: 3회 연속 피정복 시 "반란 이벤트" 발생 (방어 +20%, 1시간)
  3. **재무부 안정화 기금**: 시가총액 하락 시 Reserve(5%)에서 자동 매수

### EF-03 [HIGH] 195개 토큰 유동성 분산 문제

- **설계**: 195개 국가 토큰 각각에 $AWW 페어 유동성 풀 필요
- **문제**: D등급 47개국의 초기 발행량 5M, 유동성 풀 30% = 1.5M 토큰. 거래량 거의 0.
  - Defense Oracle이 시가총액을 읽어도 thin pool에서의 가격은 의미 없음
  - $100 매수로 가격 100x 가능 → 방어 버프 조작
- **v11 기존 지적**: H-05에서 "S등급 8국만 먼저" 제안. 하지만 195국 전체가 게임 핵심.
- **수정안**:
  1. **등급별 토큰 출시**: S(Day 1) → A(Week 2) → B(Week 3) → C/D(시즌 2+)
  2. **공유 유동성 풀**: 같은 등급 토큰들이 하나의 풀 공유 (유동성 집중)
  3. **최소 유동성 게이트**: TVL < $1,000인 풀은 Defense Oracle 미적용 (조작 방지)

### EF-04 [HIGH] 무역 수수료 + 요충지 독점 → 경제 파괴

- **설계**: 파나마/수에즈/말라카 점령 팩션 → 무역 수수료 10~15% 징수
- **문제**: 3개 요충지를 모두 점령한 단일 팩션은 전 세계 무역의 30~45% 수수료를 가져감. 이는 다른 팩션의 경제를 붕괴시키는 수준.
  - 한국↔사우디 무역: 말라카 10% + 추가 요충지 = 20%+
  - 수수료 회피를 위해 무역 자체를 포기하면 자원 부족으로 전투력 하락
- **영향**: 요충지 조기 확보 팩션이 경제적으로 압도. "Rich get richer" 극대화.
- **수정안**:
  1. 단일 팩션 요충지 최대 2개 제한
  2. 수수료 상한 5%로 하향
  3. "밀수 루트" 메커니즘: 요충지 우회 가능하지만 Oil 추가 비용 + 50% 시간 지연

### EF-05 [MEDIUM] 자원 생산 공식의 무한 성장

- **설계**: `자원 생산 = base × tier_mult × sovereignty_bonus × staking_bonus × tech_bonus`
- **문제**: 모든 보너스가 곱연산이면 후반부 기하급수적 성장:
  - 한국(A등급): 95 × 2.0 × 2.0(Lv.5) × 1.2(스테이킹) × 1.15(Tech) = **524/hour**
  - 기본값 대비 5.5배. 전투 보상 대비 자원이 과도해져 전투 인센티브 소실.
- **수정안**: 보너스를 합연산으로 변경: `base × (1 + tier_bonus + sov_bonus + stake_bonus + tech_bonus)` → 최대 ~3배 제한

## 3. 거버넌스 논리 빈틈 (Governance Logic Gaps)

### GG-01 [CRITICAL] 거버넌스 타임라인 vs 게임 속도 불일치

- **설계**: 투표 24시간 + 타임락 12시간 = 정책 변경에 **36시간** 소요
- **게임 속도**: 에포크 10분, 세션 1시간, 주권 24시간
- **문제**: 세율 변경 하나에 36시간이면, 1주일 시즌 에라 안에 정책 변경이 2회밖에 안 됨.
  - 전쟁 선포 승인(51% 찬성)은 48시간 준비 기간 전에 완료되어야 하므로, 전쟁 결정까지 36+48=**84시간(3.5일)**
  - Week 3(제국의 시대) 시작일에 전쟁 투표해도 실제 Siege는 Week 4 시작 즈음
- **영향**: 거버넌스가 게임 템포와 완전히 분리. 유저 입장에서 "투표해도 소용없다" 느낌.
- **수정안**:
  1. 투표 기간 6시간 + 타임락 2시간 = 8시간으로 단축
  2. 긴급 투표 (Emergency Vote): 팩션 리더 발동, 정족수 20%, 1시간 투표
  3. 전쟁 선포는 거버넌스가 아닌 팩션 Council 다수결(5명 중 3명)로 분리

### GG-02 [HIGH] 소국 토큰 거버넌스 독점 취약점

- **설계**: 쿼드라틱 보팅, 정족수 10% of circulating supply
- **문제**: D등급 국가의 circulating supply 계산:
  - 총 발행: 5M, treasury 50% = 2.5M, liquidity 30% = 1.5M, community 15% = 0.75M, reserve 5% = 0.25M
  - 실제 유통: community 0.75M + 유통 중 liquidity의 일부 ≈ 1M
  - 정족수 10% = **10만 토큰** → D등급 토큰 가격이 $0.001이면 $100으로 정족수 충족
  - sqrt(100,000) = 316표 → 사실상 1인 독재 가능
- **영향**: 소국 거버넌스가 매우 저렴하게 장악 가능. "거버넌스 공격" 벡터.
- **수정안**:
  1. 최소 **유니크 지갑 3개** 이상 투표 시 유효 (1인 다수 지갑은 KYC로 방지)
  2. 정족수를 고정값으로: 최소 10명 이상의 서로 다른 보유자 투표
  3. 소국은 거버넌스 비활성 → 팩션 리더가 직접 정책 결정

### GG-03 [HIGH] 이원 거버넌스 권한 충돌 미해결

- **v11 C-02에서 지적**: 팩션 리더(군사/외교) vs 토큰 홀더(경제) 역할 분리 제안
- **미해결 시나리오**:
  - 팩션 리더가 전쟁 선포 → 토큰 홀더가 세율 0%로 투표 → 전쟁 비용 조달 불가 → 교착
  - 토큰 홀더가 무역 개방 100% 투표 → 적대 팩션이 경제 침투 → 팩션 리더 항의
  - 토큰 홀더와 팩션 리더가 서로 다른 사람일 경우 (외부 투자자 vs 플레이어)
- **수정안**: 명시적 권한 매트릭스 필요:

| 정책 | 결정 주체 | 거부권 |
|------|----------|--------|
| 세율 | 토큰 투표 | 팩션 리더 거부 가능 (1회) |
| 전쟁 선포 | 팩션 Council | 토큰 홀더 66% 반대 시 취소 |
| 무역 정책 | 토큰 투표 | 없음 |
| 배치/소환 | 팩션 Commander | 없음 |

### GG-04 [MEDIUM] UN 상임이사국 거부권 남용

- **설계**: S등급 8국 지배 팩션 = 상임이사국 (거부권 보유)
- **문제**: 8개 상임이사국 중 1개만 반대해도 결의안 부결. 실제 유엔과 동일한 문제.
  - S등급 국가를 지배하는 팩션이 자기에게 불리한 결의안을 전부 거부
  - "경제 제재" 결의안을 S등급 팩션이 항상 거부 → UN이 사실상 무력화
- **수정안**:
  1. 거부권 횟수 제한: 시즌당 3회
  2. 거부권 비용: 500 Influence 소모
  3. "Override Vote": 비상임이사국 80% 찬성 시 거부권 무효화

## 4. 전투/에포크 밸런스 (Combat & Epoch Balance)

### CB-01 [CRITICAL] Peace Phase 파밍 → War Phase 압도 (선점 이점 과대)

- **설계**: Peace(5분, PvP OFF, 오브 파밍) → War(5분, PvP ON)
- **문제**: Peace Phase에서 5분간 방해 없이 레벨업한 에이전트 vs War Phase 시작 시 늦게 합류한 에이전트는 5레벨+ 차이. v14의 무기 진화(Lv.1→5)와 패시브 스택을 고려하면, Peace 파밍 선점자가 War에서 압도적.
- **추가 문제**: 리스폰 데스매치에서 사망 에이전트는 레벨은 유지하지만 **위치 랜덤 리스폰** + 속도 -30%(40틱). 이 페널티가 Peace Phase 파밍 갭을 충분히 메워주지 못함.
- **수정안**:
  1. Peace Phase에서도 약한 PvP 허용 (데미지 50%)
  2. War Phase 시작 시 "레벨 균등화" — 전체 에이전트 레벨을 평균 ±2 범위로 보정
  3. War Phase에 추가 XP 배율 ×2 (늦게 합류해도 빠르게 따라잡기)

### CB-02 [HIGH] S등급 6000×6000 아레나 에이전트 밀도 문제

- **설계**: S등급 아레나 6000×6000px, 최대 50 에이전트
- **문제**: 36,000,000px² / 50 = **에이전트당 720,000px²** (848×848px 영역). 상호작용이 극히 드묾.
  - Peace Phase에서 오브를 수집하려면 넓은 영역을 돌아다녀야 함
  - War Phase에서 적을 찾는 데 상당 시간 소요
  - 관전자 입장에서 "빈 화면"이 대부분
- **반면**: D등급 1500×1500px, 8 에이전트 = **에이전트당 281,250px²** (530×530px). 훨씬 밀도 높음.
- **영향**: 등급 간 게임 경험 격차. S등급은 지루하고, D등급은 혼잡.
- **수정안**:
  1. 아레나 크기를 에이전트 수에 동적 조절: `size = base_size × sqrt(agents / max_agents)`
  2. Shrink Phase를 모든 에포크에 적용 (Peace에서도 천천히 축소)
  3. 오브 밀도를 아레나 크기에 반비례하게 조절

### CB-03 [HIGH] 무기 5종 슬롯 + 10종 패시브 → 빌드 복잡성 폭발

- **설계**: 무기 최대 5슬롯 × 10종 + 10종 패시브 × 최대 8스택 + 10종 시너지
- **문제**: 가능한 빌드 조합 수가 천문학적. LLM 에이전트도 최적 빌드를 찾기 어려움.
  - C(10,5) × 패시브 조합 × 시너지 = 수천만 가지
  - 유저가 "빌드 프로필"을 설정해야 하는데, 어떤 조합이 좋은지 알 수 없음
  - "메타 빌드"가 발견되면 모든 에이전트가 동일 빌드 → 다양성 상실
- **수정안**:
  1. 프리셋 빌드 5~8개 제공 (초보자 가이드)
  2. "빌드 추천" AI: 현재 아레나 메타에 따라 카운터 빌드 제안
  3. 시즌마다 밸런스 패치로 메타 시프트 유도

### CB-04 [HIGH] 국가 지형 보너스 스택의 불공정

- **설계**: 한국 = Forest(은폐 -20%) + Urban(벽 엄폐 -30%) — 두 가지 테마 동시 적용
- **문제**: 일부 국가는 2개 테마가 중복 적용되어 과도한 방어 보너스. v11 §4.4에서 한국이 "Forest + Urban" 동시 태깅.
  - Forest -20% + Urban -30% = 원거리 데미지 **-50%** (사실상 반감)
  - 여기에 토큰 방어 +30% + 주권 보너스까지 스택
- **수정안**:
  1. 국가당 1개 테마만 적용 (주요 지형 선택)
  2. 테마 보너스 상한: 모든 방어/공격 보너스 합산 최대 ±25%
  3. Siege 전투에서는 테마 보너스 비활성 (공정한 쟁탈전)

### CB-05 [MEDIUM] v14 Capture Point 시스템 세부 규칙 미정의

- **설계**: War Phase에서 "캡처 포인트 쟁탈" 언급 (v14 §4.1)
- **문제**: 캡처 포인트의 위치, 수, 점령 조건, 점수 가중치가 정의되지 않음.
  - 포인트가 아레나 중앙 1개? 여러 곳?
  - 점령 시간은? 존재 시 점령? 일정 시간 머물기?
  - 캡처 포인트 점수 vs 킬 점수 비율은?
- **수정안**: 캡처 포인트 상세 스펙 정의 필요:
  - 에포크당 3개 포인트 (아레나 삼각형 배치)
  - 30초 점령 유지 시 해당 포인트 확보
  - 포인트 점수 = 킬 15점의 2배 = 30점/포인트 (킬과 점령의 균형)

## 5. 에이전트 시스템 빈틈 (Agent System Gaps)

### AG-01 [CRITICAL] LLM 2초 타임아웃 vs 20Hz 게임 틱 — 치명적 반응 격차

- **설계**: LLM 의사결정 타임아웃 2초, 게임 20Hz (50ms/틱)
- **문제**: LLM이 결정을 내리는 동안 **40틱**이 지남. 이 기간 동안:
  - 적 무기 8~40회 발사 (무기별 쿨다운 0.5~2.5초)
  - 에이전트가 600+px 이동 (300px/s × 2s)
  - 오브 5~10개 생성/소멸
  - v14 무기 시스템에서 GravityBomb, ThunderClap 같은 순간 회피가 필요한 공격은 LLM이 대응 불가
- **추가 문제**: "heading 유지 방식"은 이동만 해결. 무기 발사 타이밍, 스킬 사용, 회피 기동은 서버가 자동 처리해야 함. 그런데 이 "서버 자동 처리" 로직이 정의되지 않음.
- **영향**: LLM 에이전트가 기본 봇 AI보다 **못한** 성능. v15의 "AI 에이전트 퍼스트" 컨셉과 모순.
- **수정안**:
  1. **2계층 AI**: 서버 측 "반사 AI" (100ms, 회피/기본 공격) + LLM "전략 AI" (2초, 빌드 선택/포지셔닝/타겟)
  2. LLM은 "전략 지시"만 담당: "중앙 캡처 포인트 확보, 공격적 포지셔닝, DPS 빌드 우선"
  3. 서버가 전략 지시를 해석하여 50ms 단위 행동으로 변환

### AG-02 [HIGH] 에이전트 입력 10Hz vs 인간 30Hz — 구조적 불이익

- **설계**: Agent ≤10Hz, Human 30Hz input
- **문제**: 인간이 3배 세밀한 조종 가능. 특히 Commander Mode에서 인간이 직접 조종하면 AI 에이전트 대비 3× 정밀도.
  - v15가 "에이전트 전용 아레나"를 제공하는 것은 이 문제를 인지한 것이나, "혼합 아레나"에서는 불공정
- **수정안**:
  1. 혼합 아레나에서 인간 입력도 10Hz로 제한 (공정성)
  2. 에이전트 전용 아레나: 서버 틱 자체를 10Hz로 낮추어 반응 시간 균등화
  3. 에이전트에 "예측 이동" 기능: heading + 속도를 설정하면 서버가 곡선/궤적 보간

### AG-03 [HIGH] 에이전트 배치 거리 비용 공식 미정의

- **설계**: "배치 비용: 거리에 비례 (본국 = 무료, 원거리 = Oil 소모)" (v11 §4.3)
- **문제**: "거리"의 정의가 없음:
  - Great circle distance? (지구 표면 최단 거리)
  - 인접국 홉 수? (한국→일본 1홉, 한국→브라질 6+홉)
  - Oil 비용 공식? (1km당 0.01 Oil? 1홉당 10 Oil?)
  - 해양 횡단은 비용 추가?
- **영향**: 유저가 배치 비용을 예측할 수 없어 전략 수립 불가.
- **수정안**: `Oil_cost = base_cost × hop_count × terrain_multiplier`
  - base_cost = 10 Oil/홉
  - 해양 홉 = ×1.5
  - 본국 = 0

### AG-04 [MEDIUM] v15 ELO vs v11 팩션 점수 — 이중 평가 체계

- **설계**: v15 개인 ELO 레이팅 + v11 팩션 기반 주권 결정
- **문제**: 높은 ELO 에이전트가 약한 팩션에 속하면:
  - 개인적으로는 뛰어나지만 팩션 점수가 낮아 주권 획득 불가
  - ELO 매칭으로 강한 상대만 만남 → 킬 점수 낮음 → 팩션 기여도 하락
  - 반대로 약한 ELO가 강한 팩션에 있으면 "캐리됨" → ELO 왜곡
- **수정안**:
  1. ELO는 에이전트 전용 아레나에서만 적용 (팩션 전투와 분리)
  2. 팩션 전투는 ELO 무관, 순수 팩션 점수만
  3. 에이전트 전용 아레나 결과는 주권에 영향 없음 (순수 랭킹/보상)

## 6. 팩션/외교 논리 결함 (Faction & Diplomacy Flaws)

### FF-01 [CRITICAL] 50명 팩션 × 3명 최소 → 최대 16국 지배 vs "World Emperor" 100국 조건

- **설계**: 팩션 최대 50명, 국가 점령 최소 3명 필요, World Emperor = 100+국 지배
- **문제**: 50명 ÷ 3명/국가 = **최대 16.6국**. World Emperor(100국)는 물리적으로 **불가능**.
  - 동맹(Military Alliance)은 "공동 방어"이지 공동 주권이 아님
  - 팩션 A가 50국, 팩션 B가 50국 지배해도, 동맹이므로 한 쪽이 100국이 아님
- **영향**: 가장 야심찬 칭호가 달성 불가능. 게임 목표 자체가 논리적 모순.
- **수정안**:
  1. **팩션 인원 상한 확대**: 100~200명 (대규모 팩션 허용)
  2. **동맹 연합 주권**: Military Alliance 내 팩션들의 영토 합산으로 대륙/세계 보너스 판정
  3. **최소 점령 인원 하향**: 1명도 점령 가능, 단 3명 미만은 Sovereignty Lv.1 상한
  4. **World Emperor 조건 완화**: 50국으로 하향

### FF-02 [HIGH] NAP(불가침 조약) 스팸 방지 없음

- **설계**: Non-Aggression Pact = 100 Influence, 72시간 지속
- **문제**: 공격당할 위기의 팩션이 모든 적대 팩션에 NAP 발동:
  - 10개 팩션과 NAP = 1000 Influence (큰 비용이 아님)
  - 72시간 동안 완전 무적 (누구도 공격 불가)
  - NAP 만료 직전에 다시 갱신 가능 → 영구적 면역
- **수정안**:
  1. NAP 동시 활성 최대 3개
  2. NAP는 **쌍방 합의** 필요 (일방 발동 불가)
  3. NAP 갱신 쿨다운: 만료 후 24시간 대기

### FF-03 [HIGH] 항복 패널티의 비선형성

- **설계**: 항복 = 영토 20% 양도
- **문제**:
  - 2국 지배 팩션 항복: 20% = 0.4국 → 반올림 0국 → **패널티 없음**
  - 50국 지배 팩션 항복: 20% = 10국 → **막대한 손실**
  - 소규모 팩션은 전쟁 리스크 없이 도발 가능 (항복해도 잃을 것 없음)
- **수정안**:
  1. **최소 양도 1국** (어떤 팩션이든 항복 시 최소 1국 상실)
  2. 비율 대신 고정: "전쟁 발동 비용 환산" — 공격 팩션이 소비한 Influence + Oil의 2배를 자원으로 배상
  3. 항복 시 48시간 "전쟁 취약" 상태: 이 기간 다른 팩션도 저비용 공격 가능

### FF-04 [MEDIUM] 주말 World War Event 시간대 불공정

- **설계**: "주간 이벤트 (주말)" — 전 팩션 참여 가능, 대규모 아레나
- **문제**: 글로벌 게임인데 "주말"은 시간대마다 다름:
  - 한국 토요일 오후 2시 = 미국 금요일 밤/새벽 = 유럽 토요일 아침
  - 특정 시간대 유저가 참여 불가 → 해당 팩션 불이익
- **수정안**:
  1. 주말 이벤트를 24시간 롤링으로 (매 시간 다른 국가 세트에서 진행)
  2. 3개 시간대(아시아/유럽/아메리카)에서 각각 1회씩 (동일 이벤트 3회 반복)
  3. 이벤트 참가 시간 창(window)을 넓게: 48시간 내 아무 때나 참가

## 7. 시즌/메타 구조 결함 (Season & Meta Structure Flaws)

### SF-01 [CRITICAL] 시즌 보상 "다음 시즌 보너스" → 왕조 효과 (Dynasty Problem)

- **설계**: 1위 팩션 → "다음 시즌 시작 보너스", 에이전트 금색 이펙트
- **문제**: Season 1 우승 팩션이 Season 2 시작 보너스 획득 → Season 2 우승 확률↑ → Season 3 보너스 → ... 영구 독주
  - "모든 국가 미점령"으로 리셋되지만, 시작 보너스가 Week 1 초기 러시에서 결정적 우위
  - 기존 멤버십/팩션 구조가 유지되므로 조직력 차이 → 신규 팩션 진입장벽
- **수정안**:
  1. "시작 보너스"를 **코스메틱 전용**으로 변경 (게임플레이 영향 없음)
  2. 또는 보너스를 **시즌 2 Week 1에만** 적용 (Week 2 이후 소멸)
  3. "시드 랜덤화": 매 시즌 팩션 시작 위치를 랜덤으로 배정 (전 시즌 영토 기반 선점 불가)

### SF-02 [HIGH] Final Rush (3분 전투) 밸런스 파괴

- **설계**: Week 4 마지막 72시간 "Final Rush" — 전투 주기 단축
- **문제**: 모든 밸런스가 10분 에포크 기준으로 설계됨:
  - 무기 진화 Lv.1→5는 10분 기준 레벨 커브 (Lv.20 목표)
  - 3분이면 Lv.7~8이 최대 → 무기 Lv.2~3까지만 → 시너지 해금 불가
  - Peace 5분 + War 5분 구조가 3분에서 어떻게 되는지 미정의
    - Peace 1.5분 + War 1.5분? → Peace에서 레벨 2~3 → War에서 무의미한 전투
- **수정안**:
  1. Final Rush에서 XP 배율 ×3 적용 (짧은 시간에도 고레벨 도달)
  2. Final Rush 전용 밸런스: 시작 레벨 Lv.10, 무기 2개 장착 상태로 시작
  3. Final Rush는 에포크 대신 "서든데스" 모드: Peace 없음, 즉시 PvP, 리스폰 불가(v10 스타일)

### SF-03 [MEDIUM] 빈 국가 "부재 점령" 문제

- **설계**: 아레나 on-demand 생성, 유저가 없으면 아레나 비활성
- **문제**: 한 팩션이 D등급 국가 40개를 한 번씩만 점령하고 떠남:
  - 아무도 해당 국가에 가지 않으면 아레나 자체가 생성되지 않음
  - 지배권 도전 전투가 일어나지 않음 → 영구 점령
  - 대륙 보너스(아프리카 15국 등)를 저비용으로 확보
- **수정안**:
  1. **지배력 감쇠**: 전투 없이 24시간 경과 시 Sovereignty Level -1 (최소 Lv.0 = 미점령)
  2. **NPC 반란**: 무인 국가에 12시간마다 NPC 반란 이벤트 (방어하지 않으면 소유권 상실)
  3. **유지 비용**: 국가 유지에 Gold/Oil 소모 (지배 국가 수에 비례하여 증가)

### 8.7 핵심 원칙

1. **기획 통합 우선**: 코드 작성 전 v11/v14/v15 모순을 해소한 **통합 기획서** 작성. 이것이 Sprint 0.
2. **경제 안전 장치**: Death Spiral, 유동성 분산, 독점 방지 — 토큰 경제는 "실패 시 복구 불가"이므로 보수적 설계.
3. **시간 스케일 통일**: 게임 속도(10분 에포크)에 맞는 거버넌스/외교/전쟁 타임라인. 현실 시간 1일 = 게임 시간 ~144 에포크.
4. **에이전트 공정성**: AI와 인간의 구조적 차이(반응 속도, 연속 플레이)를 인정하고 보상하는 설계. 동등이 아닌 "공정".
5. **점진적 복잡성**: 초반(Discovery Era)은 단순, 후반(Reckoning Era)으로 갈수록 시스템 언락. 신규 유저 이탈 방지.
